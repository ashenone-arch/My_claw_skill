## 自动 Lint 检查

解析所有知识库文件（facts.md / opinions.md / conflicts.md）中的 ID 引用，执行以下 5 项自动检测：

### 1. 破引用检测

遍历所有 opinion 和 conflict 中引用的 `[F-xxx-xxx]` 和 `[T-xxx-xxx]`，验证目标 ID 在对应文件中存在。

| 检查项 | 操作 |
|--------|------|
| opinion 引用的 [F-xxx] 在 facts.md 中存在？ | 不存在 → 报告未定义引用 |
| conflict 引用的 [T-xxx] 在 opinions.md 中存在？ | 不存在 → 报告悬空引用 |
| conflict 引用的 [F-xxx] 在 facts.md 中存在？ | 不存在 → 报告待采集事实尚未写入 |

### 2. 孤立条目检测

- 检查每条 fact 是否被至少 1 条 opinion 的「支撑事实」或「推导链路」引用
- 未被引用的 fact → **孤立条目**（信息已沉淀但未被任何观点利用，需评估是否需要新增 opinion 或标注为待利用）

### 3. 悬空引用检测

- 检查每条 opinion 引用的 fact 是否标记了 `[已失效 YYYY-MM]`
- 引用了已失效 fact 的 opinion → **悬空引用**（opinion 可能已过时，需修正）

### 4. 过期修正检测

- 检查 opinion 的「修正记录」中提及的 facts 当前状态是否与修正时一致
- 如果 fact 在修正记录之后又有更新，但 opinion 未同步 → **过期修正**（opinion 可能未反映最新事实）

### 5. 稀疏主题检测

- 检查同一主题文件夹内 facts 之间是否存在交叉引用（通过「关联实体」字段判断）
- 如果同一主题下 >5 条 fact 但彼此之间无任何共享关联实体 → **稀疏主题**（信息孤岛，LLM 难以发现深层关联）

---

## 输出格式

```
【自动 Lint】

■ 破引用：✅ 全部有效 / ⚠️ X 条未定义：[列出]
■ 孤立条目：✅ 无 / ⚠️ X 条孤立 fact：[列出]
■ 悬空引用：✅ 无 / ⚠️ X 条 opinion 引用已失效 fact：[列出]
■ 过期修正：✅ 无 / ⚠️ X 条 opinion 修正记录未同步：[列出]
■ 稀疏主题：✅ 无 / ⚠️ X 个稀疏主题：[列出]
```
