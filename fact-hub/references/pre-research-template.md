# 预研框架模板 v2.0

> 步骤 2 执行前读取。v2.0：升级为三级层级知识库结构，T→O 迁移，ID 格式层级化。

---

## 输出格式

```
【金字塔预研框架】[主题]

## L1：矛盾定位 + 重要性判决

L1: [1 句话：主要矛盾 + 主要方面 + 重要性判决（行动级/观察级）]（置信度）

## L2：关键论据 + 矛盾展开

L2:
  - 矛盾对 1：[A vs B] — 主要矛盾？[是/否] — 主要方面：[A/B]
    L3: [数据锚点 1]（来源 日期）[证据等级]
    L3: [数据锚点 2]
  - 矛盾对 2：[C vs D] — 次要矛盾 — 主要方面：[C/D]
    L3: [数据锚点]

## L2：三维度验证

### 维度一：MECE 检查

L2:
  - 覆盖：✅/⚠️ 事实/真相/观点/情绪（四层无遗漏）
  - 无重叠：✅/⚠️（各子Agent 维度论证重心不重合）
  - 维度间无遗漏：✅/⚠️（[F vs T] 冲突、[事实 vs 推断] 冲突已被覆盖）

### 维度二：逻辑链完备性检查

L2:
  - 🔴/❓ 项 → 子Agent 维度：每个 🔴/❓ 是否有对应的子Agent 负责深挖？
  - 预热结论 → 矛盾定位：预研发现的争议点是否在矛盾定位中得到体现？
  - 步骤 1 检索 → 已知数据包：每条检索结果是否分配到了正确的维度数据包？

### 维度三：论点对齐检查

L2:
  - L1 首席判断是否有各维度论据支撑？✅/⚠️
  - 风险反向论点是否在子Agent 维度中有对应的验证方向？✅/⚠️
  - 对首席判断的"支撑/修正/推翻"预期是否在每个维度塔尖中有锚点？✅/⚠️

## L2：反转追问

L2: 市场共识是 [1 句话描述]
  - 共识可能在 [维度] 上出错
    L3: [为什么]
  - 推翻当前判断的证据：[具体可观测信号]
    L3: [触发条件]

## L2：塔尖断言（必做）

> 为每个选定的子Agent 维度预写一句断言句。子Agent 只能展开论证，不能推翻。

L2（塔尖断言）:
  - 地缘维度塔尖：[1 句断言，如 "台海安全焦虑已将半导体供应链从效率优先转向安全优先，这一转变不可逆"]
  - 产业维度塔尖：[1 句断言]
  - 技术维度塔尖：[1 句断言]
  - 财务维度塔尖：[1 句断言]

**写作规范**：
- 每句用断言句，不用主题描述句。❌ "地缘政治维度分析" ✅ 断言句
- 各维度塔尖之间逻辑自洽，不相互矛盾
- 如果某维度数据不足无法形成断言，标注 `[待数据补充]`

## L2：子Agent 分配方案

L2: 需启动的子Agent：[A地缘/B产业/C技术/D财务]（根据 🔴/❓ 涉及的维度选择）

L3（已知数据包 — 簇简报格式，按维度）:
  - **簇 A 简报（地缘维度）**：
    塔尖：[1 句]
    知识库引用：[F-xxx-xxx] [O-xxx-xxx]
    步骤 1 检索数据：[条目列表]
    文章原文相关内容：[条目列表]

  - **簇 B 简报（产业维度）**：
    塔尖：[1 句]
    知识库引用：[F-xxx-xxx]
    步骤 1 检索数据：[条目列表]
    文章原文相关内容：[条目列表]

  - **簇 C 简报（技术维度）**：
    塔尖：[1 句]
    知识库引用：[F-xxx-xxx]
    步骤 1 检索数据：[条目列表]
    文章原文相关内容：[条目列表]

  - **簇 D 简报（财务维度）**：
    塔尖：[1 句]
    知识库引用：[F-xxx-xxx]
    步骤 1 检索数据：[条目列表]
    文章原文相关内容：[条目列表]
```

## L2：层级检查（v2.0 新增）

> 当前分析处于哪个层级（大行业 / 细分行业 / 具体公司）？是否有跨层事实被遗漏？

L2（层级检查）:
  - 当前层级：[大行业 / 细分行业 / 具体公司]
  - 是否有上级层级的 opinions 可作为背景约束？[列出相关 O-xxx ID]
  - 是否有下级层级（细分行业/公司）的关键事实未被纳入分析？[列出待补充条目]
  - 跨层引用是否标注了来源层级？[检查所有引用]
