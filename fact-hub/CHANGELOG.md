# fact-hub 版本历史

> 详细日志见 `SKILL.md` 中各版本的执行铁律和方法论正文。

- **v2.0**：知识库架构革命——(1) 三级层级结构：从平铺主题（4个）升级为"大行业→细分行业→具体公司"三级树形结构（3大行业/9细分/18公司）；(2) 公司级合并为 knowledge.md（facts+opinions 同文件），写入前必须通过 iFinD/alphaengine 研报补充搜索；(3) Opinion 标记 T→O（T1→O1, T2→O2），证据等级体系同步更新；(4) ID 格式层级化：`[F-AI-基础模型-Anthropic-001]` 全路径格式；(5) 步骤 0a 读取策略改为层级渐进读取（不总全量加载）；(6) 步骤 5b 冲突扫描改为沿引用链向上追溯；(7) 步骤 6 Lint 新增层级归属检查；(8) 新增跨层引用规则——facts/opinions 可引用任意层级条目；(9) 全局 README 升级为认知地图（含跨层关系图）；(10) 所有 127 条 facts + 36 条 opinions + 11 条 conflicts 自动迁移重新归类。旧文件保留未修改。
- **v1.11**：借鉴 Marko Papic《Geopolitical Alpha》约束框架方法论，对子Agent A（地缘政治）进行三层改造——(1) 约束扫描前置：三角分析前新增"约束扫描五问"（政治/经济/金融/地缘/制度），要求对每个关键行为体逐一扫描物质约束；(2) 诊断性分层与支点约束：输出新增"约束-偏好诊断表"和"支点约束"识别，明确区分高诊断性约束 vs 低诊断性偏好；(3) 偏差自检+定价对照：新增"基本归因谬误自检"和"定价对照"两个必填输出块。核心原则：约束优先于偏好（Constraints before Preferences）。子Agent B/C/D 及通用规范不受影响。
- **v1.10**：文件编辑锚点升级为强制3步检查流程——写入前必须 (1) read尾部20行 → (2) grep验证唯一性 → (3) 优先用ID做锚点，禁止用泛化字段标签（如`证据等级: F2`）；新增大追加分批策略（超3条事实/1500字符必须拆分，每批独立锚点）。陷阱速查表新增2条。基于v1.9实战中`old_string`不唯一导致facts.md插入错位的教训
- **v1.9**：三项交互与鲁棒性升级——用户显式选择子Agent维度（步骤 2 增加 AskUserQuestion 多选确认，主Agent 推荐+说明理由）；sub-sub-agent 拆分机制（12 个子维度可选，任务过重时自动建议拆分）；超时接管协议（四条完整流程——超时判定→接管流程→降级策略→后处理，主Agent 接管兜底）。步骤 0b 显式标注 AskUserQuestion 逐项确认。陷阱速查表新增 4 条。
- **v1.8**：基于 Cerebras 实战复盘的三项修复——文件编辑锚点规范（`old_string` ≤200字符，追加优先用短锚点）、子Agent 分工优化（技术精细推算→主Agent直处、广域交叉验证→子Agent）、步骤 5b 冲突裁决扫描（新 facts/opinions 写入后自动检查冲突触发条件）
- **v1.7**：借鉴 equity-deep-research"第二步并行取数"方法论，新增 facts 写入前 iFinD/alphaengine 交叉验证环节。新增：交叉验证规范（铁律#10、工具选择矩阵、验证结果标注）、data-collection-guide v1.5（mcporter call 命令模板、按事实类型选择验证工具）。半快速通道/轻量通道/步骤 5 均增加交叉验证步骤。陷阱速查表新增 2 条。
- **v1.6**：借鉴 equity-deep-research 等 5 个本地 Skill 进行系统性方法升级。新增：时间精度规则、数据源分工原则、数据优先级（P0/P1/P2）、研究预热（Broad→Narrow）、写作前综合检查（5 条）、塔尖断言机制、反向探测、簇简报格式、快速通道/半快速通道/轻量通道三级路由、中间文件清理。升级：MECE 自检→三维度验证（MECE+逻辑链+对齐）、溯源标注→三要素（证据等级+来源工具/文件+数据时间）、冲突升级展示（F1 vs T2 不自动裁决）。陷阱速查表新增 6 条。
- **v1.5**：步骤 0 拆为 0a（分析对照）+ 0b（决策路由）两步摄入；新增增量缓存（跳过未变更文件）；新增 log.md 操作日志；步骤 6 升级为自动 lint（破引用/孤立条目/悬空引用检测）；README.md 扩展为 LLM 导航入口含 opinion 摘要。
- **v1.4**：新增"写入规范"章节：facts 只追加不删除 + falsification 检测流程 + yyyy-mm 日期标注 + truths 多跳链式推导 + 修正记录三段式 + conflicts 待采集事实闭环 + facts 影响真相反向索引（可选）。
- **v1.3**：新增快速对照层（步骤0），知识库优先消除重复检索；全流程输出改为金字塔三层 bullet point 结构；子Agent 从2个拆为4个独立维度（地缘/产业/技术/财务），消除检索冗余，主Agent预分配"已知数据包"。
- **v1.2**：重构证据等级体系，新增二级升一级逻辑，明确 facts/opinions/conflicts 三层职责分工，opinions 更新须用户逐条确认。

- **v1.0**：初版。基于辩证研究舱框架重构，新增版本标记、证据等级体系、铁律体系、并行架构、陷阱速查表、Acceptance Self-check、MECE 质量检查。

- **v2.1**：文件写入安全性升级（基于 v2.0 实战踩坑）——(1) 新增执行铁律 #11「尾部追加用 bash」：步骤 5 写入时先判断场景——尾部追加新条目 → 用 bash `cat >>` heredoc（零锚点风险）；文件中间修改 → 用 edit（走 v1.10 锚点规范）；(2) 写入规范重构为「文件写入工具选择（v2.1 强制）」：明确定义"尾部追加 vs 中间修改"两种场景的工具选择矩阵，新增 bash heredoc 模板；(3) 新增「git 安全护栏」：写入前 git status 确认工作区干净、磁盘文件远大于 git 版本时先 git stash、写入后验证行数、异常时 git checkout 回滚；(4) 陷阱速查表新增 3 条（🔴 edit 用于尾部追加 / 🟡 写入前未 git status / 🟢 git checkout 丢未提交内容）。基于步骤 5 连续 3 次写入失败根因分析——锚点 `证据等级: F2+` 在 220 行文件中匹配到第 30 行而非末尾、git checkout 将 220 行文件恢复到 45 行旧快照。

- **v2.4**：写入安全重构——(1) 铁律 #8 重写为"安全写入"：尾部追加从 heredoc 改为 write（read→拼接→整写），edit 锚点强制以 `[F-/O-/C-` ID 开头；(2) 4.3「写入规范速查」升级为 6 条规则表，每条附"违反后果"原因说明；(3) git 护栏简化：写入前 git stash → 写入后验证 → 异常 stash pop 回滚；(4) 新增写入后 grep 验证要求；(5) 退出前自检新增第 5 项"grep 验证写入内容"；(6) 踩坑日志新增 heredoc 静默失败事故；(7) references/writing-standards.md 同步更新——废弃 heredoc，工具选择矩阵从 3 列改为 4 列（含步骤列）
