## 段8 · 跟踪指标

**所需数据**：AlphaClaw 公告 + iFinD 分红/事件

**核心假设**：我们假设 [下次财报最大的超预期点在X，最大的低于预期风险在Y]。三个月内 [事件A/B/C] 最可能触发股价波动。若 [X未实现 / Y发生]，需相应调整持仓判断。

**写作模板**：

```
### 下次财报重点看
- [指标名]：[当前值] → [关注方向]（F2-iFinD get_stock_financials）
  触发条件：[若实际值 < X / > Y，则 [行动]]
- [指标名]：[当前值] → [关注方向]
  触发条件：[若实际值 < X / > Y，则 [行动]]

### 三个月内关键变量
1. [变量名]
   - 跟踪方式：[怎么查]
   - 触发条件：[X则Y]（C1）
   - 数据来源：[工具名]
   - 优先级：[高/中/低]

2. [变量名]
   - 跟踪方式：[怎么查]
   - 触发条件：[X则Y]（C1）
   - 数据来源：[工具名]
   - 优先级：[高/中/低]

3. [变量名]
   - 跟踪方式：[怎么查]
   - 触发条件：[X则Y]（C1）
   - 数据来源：[工具名]
   - 优先级：[高/中/低]

### 事件日历
| 事件 | 预期日期 | 关注要点 | 数据来源 |
|------|---------|---------|---------|
| 分红派息日 | [日期] | 股息率、除权影响 | F2-iFinD get_stock_financials |
| 大股东解禁 | [日期] | 解禁规模、减持意愿 | F2-iFinD get_stock_events |
| 业绩会/投资者交流 | [日期] | 管理层指引变化 | M1-AlphaClaw search_finance_reports |
| 行业政策/展会 | [日期] | 行业催化 | AlphaClaw 新闻/纪要 |
```

**写作规范**：
- 禁止"关注政策变化"等废话，必须具体到可操作的指标和触发条件
- 每个变量要有明确的跟踪方式、触发条件、数据来源和优先级

**段内综合判断**：

```
### 段8 综合判断
**[一句话：下次财报的关键观察点是X（可能超预期）和Y（可能低于预期），三个月内最可能触发股价波动的是Z（优先级：高）]**
→ 数据支撑：[引用跟踪指标表中的关键触发条件]
→ 对投资判断的影响：这些跟踪指标构成对首席投资论点的动态验证体系。若任一高优先级触发条件被激活，应在24小时内重新评估持仓判断。
```

> **衔接段9**：本段的跟踪指标中仍无法覆盖的数据需求（如内部经营数据、客户验证等），在段9 中作为待补资料列出。

## 段9 · 仍需补的资料

**核心假设**：我们假设 [前8段中已标注[需补]的数据是当前分析的主要不确定性来源]。若 [关键缺失数据获取后与当前推断矛盾]，则相关段落的结论需要修正。

**写作模板**：

```
**必需**（影响核心判断）：
- [缺失数据] — 应从 [数据源] 获取。若为 [情况A]，则段[X] 的 [结论] 成立；若为 [情况B]，则该结论需修正。
- [缺失数据] — 应从 [数据源] 获取

**建议**（提升分析质量）：
- [建议获取的数据] — 应从 [数据源] 获取

**不确定但影响判断**：
- [假设性问题] — 需 [验证方式]
```

**写作规范**：
- 此段不能为空
- 每条缺失数据必须列出获取路径
- 新增：关键缺失数据需说明"若获取后的两种情况分别意味着什么"
