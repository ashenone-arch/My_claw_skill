# 股票深度研究（9 段框架）

## 概述

对 A 股上市公司进行结构化深度研究，输出 9 段投研素材包。框架本身与数据源无关，优先使用 iFinD MCP 获取结构化财务与量化数据，用 AlphaClaw 获取定性分析（研报/纪要/公告）。

**执行模式**：两步确认 — 预研后先输出轻量框架请你确认，再深度写报告。

## 时间精度规则

> 在构造所有数据查询（iFinD / AlphaClaw）时，必须根据用户的时间表述选择正确的精度。

| 用户表述 | 查询精度 | iFinD time_start/end | AlphaClaw date_range |
|---------|---------|---------------------|---------------------|
| "今天/盘后/盘中/这两天" | 日级别 | 当日日期 | `past_day` |
| "本周/这周/过去一周" | 周级别 | 本周一 ~ 当日 | `past_week` |
| "本月/这个月/近一个月" | 月级别 | 本月1日 ~ 当日 | `past_month` |
| "最近/近期/最新"（无明确时间单位） | 季度级别 | 近3个月 | `past_quarter`（默认） |
| "本季度/近三个月" | 季度级别 | 当前季度起始 | `past_quarter` |
| "近半年" | 半年级别 | 近6个月 | `past_half_year` |
| "近一年/去年" | 年级别 | 对应年份 | `past_year` |

**原则**：
- 用户说了具体时间单位 → 必须用对应精度，禁止扩大范围
- 用户说"今天" → 必须用 `past_day`，禁止传 `past_week`
- 为了"多召回一些"而扩大时间范围是错误的——召回量由 `recall_num` 控制

## 9 段结构

1. **公司定位**：产业链位置 / 产品结构 / 差异化 vs 同业
2. **业务拆分**：各业务线收入/占比/YoY/毛利率
3. **收入驱动**：量/价/结构/客户四维拆解
4. **利润驱动**：盈利能力 / 弹性分析 / 一致预期
5. **核心竞争力与主要风险**：逐项 vs 可比公司 + 量化支撑
6. **市场焦点**：催化逻辑 / 核心担忧 / 预期差
7. **可比与差异**：关键指标横向对比（≥3家）
8. **跟踪指标**：下次财报重点 / 三个月关键变量
9. **仍需补的资料**：必需 / 建议 / 不确定但影响判断

## 执行步骤

### 第一步：研究预热（Broad → Narrow）

**目的**：在正式调数据之前，快速建立对这家公司的整体认知，确定重点关注维度。

并行执行以下操作：
1. 调用 `search_news` 查询该公司近 3 个月的新闻，了解当前市场焦点
2. 调用 `search_trending_news` 查询近期热门新闻（含情绪方向）
3. 根据新闻结果，识别该公司当前最核心的争议点（催化逻辑 vs 市场担忧）

**预热结论格式**：
```
【研究预热结论】
- 当前市场最关注：[核心议题]（来源：[新闻日期]）
- 核心担忧：[市场担心的点]
- 建议重点深挖：[1-2个需要重点验证的维度]
```

预热完成后进入第二步。如果预热发现公司近期无重大事件或新闻稀少，继续进入第二步（不阻塞）。

### 第二步：提取证券信息 + 并行取数

从用户消息中提取公司名称和证券代码。如果用户只给了公司名没给代码，用 `iFinD search_stocks` 查询。

> **执行前**：读取 `references/data-commands.md` 获取 iFinD bash 命令模板和 AlphaClaw 参数表。
> 数据源不确定时 → 读取 `references/quick-ref.md` 的"工具快速参考"表。

**同时在同一条消息中并行执行所有以下命令**（不需要等一个返回再调下一个，全部同时发出）。将 `<公司名>`、`<今日日期>`、`<近3个月日期>` 替换为实际值后，在 `D:\Alphaclaw\` 目录下执行 `references/data-commands.md` 中的全部命令模板：iFinD 股票数据 7 条 + iFinD 新闻 2 条 + AlphaClaw 定性分析 4 条，共 13 条并行发出。

> **数据优先级说明**：P0 = 缺失则报告无法完成；P1 = 重要增强；P2 = 有则更好。

### 第三步：生成预研框架，发给你确认

> **执行前**：读取 `references/framework-confirm.md` 获取预研框架模板。

按 `references/framework-confirm.md` 中的模板生成轻量预研框架，内容包括：

- 研究预热结论（第一步的输出）
- 数据获取概况（成功/缺失）
- 初步核心结论（已经可以得出的关键判断）
- 建议的可比公司名单（≥3家，附选择理由）
- 当前市场核心争议（催化逻辑 vs 市场担忧）
- **询问你**：
  1. 研究重点是否需要调整？
  2. 有没有要补充的数据或要换的可比公司？
  3. 有没有特别关注的问题希望在报告里重点分析？

**等你回复确认后，再进入第四步。**

### 写作前综合检查（确认后、动笔前）

> 确认后、动笔前，先自问以下 5 条。任一条为"否"则先补充检索，不急于动笔。

| # | 检查项 | 通过标准 |
|---|-------|---------|
| 1 | 多角度覆盖 | 已从 ≥3 个角度检索（财务/研报/新闻/行业/可比）？ |
| 2 | 核心数据就位 | P0 级数据覆盖率 ≥ 80%？段1-段7 每个关键结论有 F2 级数据锚点？ |
| 3 | 正反观点完整 | 同时检索了正面催化逻辑和负面担忧/质疑观点？ |
| 4 | 核心争议可回答 | 预研阶段发现的核心争议点有数据可以回答？ |
| 5 | 信息时效合格 | 财务数据来自最新财报期？研报/纪要来自近3个月？ |

→ **任一条未通过**：先补充检索，不急于动笔。全部通过后进入第四步。

### 第四步：生成完整 9 段报告

> **执行前**：读取 `references/analysis-template.md` 获取9段模板和证据等级规范。
> 遇到框架选择困难 → 读取 `references/framework-toolkit.md`。
> 遇到数据获取异常 → 读取 `references/troubleshooting.md`。

你确认后，按 `references/analysis-template.md` 的 9 段结构生成完整报告。关键要求：

- **数据溯源三要素**：每条结论必须同时标注证据等级（F1/F2/M1/C1/H1）+ 来源（工具名或文件）+ 数据时间（哪期财报）
- **每段标注证据等级**
- **段内采用金字塔结构**：段落小标题 = 塔尖论点 → 2-3个支撑性子论点 → F2/M1级数据锚点。禁止在子论点层面出现无数据支撑的纯判断句
- **段5 必须指名道姓对比**：杜绝"行业领先"等空泛表述。竞争维度用VRIO量化对比表呈现，风险维度写出触发条件
- **段6 催化/担忧/预期差三段内部均用金字塔**：每段末尾加一行 **Synthesis**（一句话总结该维度结论）
- **段6 可结合行情数据验证市场情绪**
- **段8 跟踪指标必须具体可操作**：包含硬触发条件（如"单季净利润 < X亿 则下调盈利预测"）
- **段9 不能为空**
- **执行 Acceptance Self-check**（模板末尾）

### 第五步：保存并交付

将报告保存为 Markdown 文件到以下地址：

**保存路径规则**：
```
D:\Alphaclaw\Output\{公司名}-{证券代码}\{报告日期YYYYMMDD}-{最新一季财报日期}.md
```

**具体规则**：
1. **子文件夹**：先在 `D:\Alphaclaw\Output\` 下创建以 `公司名-证券代码` 命名的子文件夹（如 `兆易创新-603986`）
2. **文件名**：`{报告日期}-{最新一季财报日期}.md`，其中最新一季财报日期格式为 `YYYYQ{N}`（如 `2026Q1`、`2025Q3`）
3. **覆盖规则**：同一公司、同一财报期若重复研究，**追加内容而非覆盖**，在文件末尾附上新的研究日期和更新内容（保留历史版本）

**示例**：
- 首次研究：文件名 → `D:\Alphaclaw\Output\兆易创新-603986\20260507-2026Q1.md`
- 后续跟踪（同一季度更新）：文件名 → `D:\Alphaclaw\Output\兆易创新-603986\20260508-2026Q1.md`（追加到同一文件，或另建 `20260508-2026Q1_update.md`）
- 跨季度研究：新文件 → `D:\Alphaclaw\Output\兆易创新-603986\20260820-2026Q2.md`

**交付时**：同时告知你文件夹路径和文件名，方便后续查找。例如：`报告已保存到 D:\Alphaclaw\Output\兆易创新-603986\20260507-2026Q1.md`

## 证据等级

| 标签 | 含义 | 示例 |
|------|------|------|
| F1 | 公开可验证事实 | 交易所披露 |
| F2 | 财报/金融数据库/公告 | iFinD 数据、年报 |
| M1 | 市场一致预期 + 管理层公开发言 | 卖方预测、业绩会发言 |
| C1 | 合理推演 | 需附推演链路 |
| H1 | 待核验假设 | 必须标"未经核验" |

**约束**：没有证据等级不能写。关键结论必须有 F1/F2 支撑。管理层公开发言 = M1，不是 F2。

## 数据源分工原则

> **推荐调用方式**：所有 iFinD MCP 接口优先使用 `mcporter call`（`tool` 参数 = `"服务器.工具"` 点号连接格式），详见 `references/data-commands.md`。bash + call.py 仅在 mcporter 不可用时作为备选。

| 数据类型 | 工具 | 调用方式 | 说明 |
|---------|------|---------|------|
| 结构化财务、估值、风险、股权、事件、行情 | `hexin-ifind-ds-stock-mcp.*` | `mcporter call tool="hexin-ifind-ds-stock-mcp.<方法>"` | 全量覆盖 AlphaClaw 定量查询 |
| 行业宏观（产销量/价格） | `hexin-ifind-ds-edb-mcp.*` | `mcporter call tool="hexin-ifind-ds-edb-mcp.<方法>"` | get_edb_data |
| 新闻、公告 | `hexin-ifind-ds-news-mcp.*` | `mcporter call tool="hexin-ifind-ds-news-mcp.<方法>"` | 时效性优于 AlphaClaw 研报库 |
| 研报观点、纪要、公告原文 | `search_finance_reports`（AlphaClaw） | 直接调用 | iFinD 不覆盖卖方观点 |

**禁止**：用 AlphaClaw 定量查询替代 iFinD 获取结构化数据。

## ⚠️ 陷阱速查表

> 详细故障排查 → 读取 `references/troubleshooting.md`

| 严重度 | 陷阱 | 正确做法 | 违反后果 |
|-------|------|---------|---------|
| 🔴 致命 | 管理层公开发言标为 F2 | 业绩会/交流会/管理层公开发言 = M1，不是 F2 | 证据等级污染，结论可信度归零 |
| 🔴 致命 | 可比公司 < 3 家 | 段7 至少 3 家，不足时用 `search_stocks` 补充 | 段7 无横向参照，分析缺乏锚点 |
| 🔴 致命 | 自行估算关键数字 | 数据缺失标注 `[需补]`，段9 列出获取路径 | 幻觉数据侵蚀报告可信度 |
| 🟡 重要 | 段5 纯定性描述（"行业领先""行业龙头"） | 必须引用 Beta/股权集中度/分红等 F2 级数据做量化对比 | 竞争力分析无量化支撑 |
| 🟡 重要 | 段8 写"关注政策变化"等废话 | 必须具体到指标 + 硬触发条件（如"单季净利润 < X亿 → 下调预测"） | 跟踪指标无操作价值 |
| 🟡 重要 | 跳过第三步确认机制 | 完成第二步后先输出预研框架请用户确认，用户说"直接出报告"才可跳过 | 方向性偏差风险 |
| 🟡 重要 | 行业宏观数据缺失导致段3 空泛 | 用 `get_edb_data` 补充产销量/价格指数数据 | 收入驱动分析缺乏宏观锚点 |
| 🟢 注意 | 研报摘要不足仍写报告 | 用 `get_document_detail` 获取全文 | 段5/段6 分析深度不足 |
| 🟢 注意 | iFinD 工具用裸函数名 | 必须用完全限定名 `服务器名.工具名`（如 `hexin-ifind-ds-stock-mcp.get_stock_financials`） | 工具调用失败 |
| 🟢 注意 | 段9 留空 | 基于前 8 段判断，必须列出缺失数据及获取路径 | 数据缺口不可见 |
