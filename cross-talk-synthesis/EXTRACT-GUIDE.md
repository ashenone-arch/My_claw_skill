# Sub-agent 提取模板

步骤 2 和步骤 4 的 sub-agent 启动后，**先读本文件获取格式规范**，再处理主 agent 指定的文件列表。

---

## 步骤 2：轻量内容扫描

### 提取字段

对每篇文件，只提取以下 5 个字段（不做详细观点摘录）：

| 字段 | 说明 |
|------|------|
| `title` | 文件名（去日期和后缀的语义名称） |
| `date` | 文件日期 |
| `expert_background` | 受访者/信息来源背景（一句话，注明其前/现任公司与职级） |
| `topic_tags` | 话题标签（3-8 个自然语言短语），按主 agent 指定的关注方向侧重 |
| `one_liner` | 一句话摘要（核心内容或最有价值的观点） |

**禁止**：提取 `key_points` / 详细观点摘录 / 长段落引用，这些保留到步骤 4。

### 输出格式

```markdown
### {title}
- **日期**：{date}
- **嘉宾背景**：{expert_background}
- **话题标签**：{topic_tags}
- **摘要**：{one_liner}
```

### 分组扫描附加输出

若主 agent 要求扫描一组文件，在本组全部文件扫描完成后，**额外输出**：

```markdown
### 本组话题频次汇总
| 话题 | 覆盖文件数 |
|------|-----------|
| ... | ... |
```

---

## 步骤 4：深度提取

### 核心原则

1. **塔尖候选优先**：输出开头先给该话题的「塔尖候选」——一句可验证的核心判断，单独成行。这句话将供主 agent 在 5a 快速扫读（不读全文）来定架构。写塔尖候选时：用断言句、标注是共识还是分歧、不超过 50 字
2. **证据分级**：每条信息标注 Q1/Q2/Q3（定义见下）
3. **文件溯源**：每条标注来自哪个文件（绝对路径 + 日期）
4. **分歧呈现**：如有分歧必须标注【歧见】，如有共识如实说明——不硬凑共识
5. **不丢信息**：宁可多提取不要漏。所有与话题相关的段落都应纳入

### 证据分级标注格式

| 等级 | 含义 | 标注格式 |
|------|------|---------|
| **Q1** | 直接引语 / 可验证事实 | 带引号保留原文，后注 `（[嘉宾名]，[文件日期]）` |
| **Q2** | 归纳转述 | `据[嘉宾名]在[文件]中所述` 开头，基于原文概括 |
| **Q3** | 跨文件交叉推断 | 标注推理前提 `[推理前提：...]` + 推理链路 |

**铁律**：Q3 不得以 Q1 口吻呈现。

### 输出格式

```markdown
# {话题名} 深度提取

## 塔尖候选（供主 agent 架构用）
> [一句断言式核心判断，≤50 字，标注共识/分歧类型]

## 一、{子维度 1}
- [Q1] "原文引语..."（[嘉宾名]，[文件日期]）
  来源：[绝对路径]
- [Q2] 据[嘉宾名]在[文件]中所述，...

## 二、{子维度 2}
...

## 分歧标注
| 分歧点 | A 方观点 | B 方观点 | 分析 |
```

---

## 通用要求

- **不合并文件处理**：每个文件独立读取，不将多文件合并为单文本再处理
- **并行读取**：同一组内的文件可以并行 read
- **输出直达**：扫描/提取完成后直接输出结果，不加"任务完成"等结束语
