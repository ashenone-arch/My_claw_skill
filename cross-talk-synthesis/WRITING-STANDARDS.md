# 文章撰写标准

以下标准适用于 cross-talk-synthesis 最终文章输出。

## 保真底线

- 不凭空编造任何事实、数据或观点；不歪曲任何嘉宾的立场和原意
- 允许在同一段内对纯粹的口语重复进行压缩合并，但不削去携带新信息维度的表述
- 跨文件引用同一话题的表述时，忠实保留每位嘉宾的原话质地，不因归纳而丢失差异性

## 信息增量测试

判断一段话是否可以压缩：

| 信号 | 动作 |
|------|------|
| 携带新事实数据 | 保留 |
| 添加新分析维度 | 保留 |
| 增加限定条件 | 保留 |
| 给出因果解释 | 保留 |
| 有具象化场景/例子 | 保留 |
| 纯粹换措辞复述、口语填充、自我修正中间表述 | 可压缩 |
| **兜底原则**：拿不准时 | **默认保留** |

## 主持人话语处理

- 原文中主持人功能性互动（附和、简短过渡）→ 去除
- 内容性提问/评论 → 转为叙事引导句，避免 Q&A 格式

## 叙事性 vs 说明性双轨

- **叙事性内容**（嘉宾亲述的个人经历、关键转折时刻、情绪高峰）→ 使用直接引语，保留冲击力
- **说明性内容**（概念解释、背景、系统原理）→ 精炼书面语准确归纳，受信息增量测试约束

## 极简过渡句

- 话题之间可加 1-2 句过渡，仅用于桥接上下文的跳跃
- 不凭空编造话题间关系

## 专有名词标注

- 人名、公司名、产品名、技术术语、特定概念在首次出现时用中文后括号注明英文原文
- 后续同一名词可只写中文

## 行文风格

- **bullet point 优先**：主体内容使用 bullet point 短句，每个信息点 2-3 行内讲清
- **段落体限引言/结语**：仅引言和结语允许使用连续段落体，正文均用列表驱动
- **每点独立可扫**：读者扫一眼 bullet 就能抓住信息点，不需通读段落
- **短句原则**：一个 bullet 只承载一个信息维度；如需展开，用子级 bullet 缩进

## 输出格式禁止

- **禁止生成图片/图表/可视化**：全文仅用 markdown 原生格式（文字、表格、列表）
- **表格替代图表**：需要对比数据时用 markdown 对比表格，不生成柱状图/折线图/饼图
- **列表替代流程图**：需要展示流程/层级时用嵌套列表，不生成流程图/思维导图
- **禁止 ASCII art**：不使用字符画、分隔线装饰等非内容元素

## 多篇特有规则

- 每个话题段落开头必须注明嘉宾姓名，让读者知道谁的发言
- **观点分歧必须呈现**——不为了叙事流畅而让"各方似乎达成共识"。如果原文中观点确实一致，也如实说明"A 和 B 在这一问题上观点一致"
- **不添加原文中没有的"分析""评价""总结"**——文章的价值在于把多篇原文中的共通脉络展示出来，而非代替读者做判断

## 信息来源标注规范

- 每个子话题开头用 `**来源覆盖**：[嘉宾A-文件1]、[嘉宾B-文件2]` 标注信息来源
- 关键数据后用括号标注来源和日期，如"（TME商业化负责人，2026年2月）"
- 避免在文中使用"据采访显示"等模糊表述，应具体到哪位嘉宾、哪份文件

## 证据分级与写作规范

文章中的所有观点必须按以下三级标注，与 SKILL.md 中的证据分级体系一致：

| 等级 | 含义 | 写作规范 |
|------|------|---------|
| **Q1** | 直接引语 / 可验证事实 | 必须用引号包裹、标注来源文件路径和日期。示例：`"具身智能的落地场景远少于预期"（商汤前高管，conversation-20260115.md）` |
| **Q2** | 归纳转述 | 以"据[嘉宾名]所述"开句，保留原意但不逐字引用。落笔前确认：这句话能否在原文中找到对应段落？ |
| **Q3** | AI 交叉推断 | 用于"交叉分析"小节。必须标注推理前提（基于哪些 Q1/Q2 事实）和推理链路（从前提如何得出判断）。禁止以"嘉宾们认为"包装 Q3 推断。 |

**跨级禁止**：
- Q3 不得以 Q1 口吻呈现——"嘉宾们一致认为 X"仅在每个嘉宾的原文中可独立验证 X 时才成立
- Q2 不得混入主观延伸——归纳转述限于原文已有内容，不添加原文没有的评价

## 金字塔写作要求

每个选定话题的撰写必须遵循金字塔结构（强制，由 SKILL.md 步骤 5 约束）：

```
塔尖（核心判断：1 句）
  ├─ 嘉宾 A 的观点 → Q1/Q2 锚点（具体引语或归纳 + 来源）
  ├─ 嘉宾 B 的观点 → Q1/Q2 锚点
  └─ 交叉分析 → Q3（标注推理链路）
```

**塔尖写作原则**：
- 先判断是共识还是分歧——不能模棱两可
- 共识 → 写出共同认可的具体内容及认可边界
- 分歧 → 写出分歧的根因（信息差？立场差？时间差？）
- 元主题 → 识别多篇共同指向但嘉宾未明确说出的深层命题

**反模式**（以下写法视为不合格）：
- ❌ "A 认为 X。B 认为 Y。两人观点不同。"（平铺，无塔尖）
- ❌ "A 和 B 在 X 上达成了共识。"（Q3 包装为 Q1，未标注推理链路）
- ✅ "A 和 B 对 X 的分歧根源在于时间尺度——A 关注 1-2 年的短期渗透率（Q1-[文件A]，'2027年前不会超 5%'），B 关注 10 年级别的技术替代逻辑（Q1-[文件B]，'最终替代率可达 80%'）。两者在核心假设上无实质冲突，但因视角尺度不同得出不同结论。"（Q3-[推理前提：两条 Q1 的时间尺度不同，核心假设无冲突]）

## 分层撰写规则

当用户只选择了部分话题做深度展开时：

- **选定话题**：深度展开，篇幅与原文讨论深度正相关。逐嘉宾/逐维度对比，带 Q1 原文引语和来源标注。用 bullet point 短句组织
- **非选定话题**：归入结尾「其他涉及话题」章节，每话题 1-2 段简要提及，总计不超过全文 20%
