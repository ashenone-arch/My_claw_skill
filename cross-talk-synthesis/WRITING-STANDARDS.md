# 文章撰写标准

以下标准适用于 cross-talk-synthesis 最终文章输出。

<!-- full: 快速通道和标准模式（≤3 话题）时 make-report 读取全文件 -->

## 保真底线

- 不凭空编造任何事实、数据或观点；不歪曲任何嘉宾的立场和原意
- 允许在同一段内对纯粹的口语重复进行压缩合并，但不削去携带新信息维度的表述
- 跨文件引用同一话题的表述时，忠实保留每位嘉宾的原话质地，不因归纳而丢失差异性

## 信息增量测试

判断一段话是否可以压缩：

| 信号 | 动作 |
|------|------|
| 携带新事实数据 | 保留 |
| 添加新分析维度 | 保留 |
| 增加限定条件 | 保留 |
| 给出因果解释 | 保留 |
| 有具象化场景/例子 | 保留 |
| 纯粹换措辞复述、口语填充、自我修正中间表述 | 可压缩 |
| **兜底原则**：拿不准时 | **默认保留** |

## 主持人话语处理

- 原文中主持人功能性互动（附和、简短过渡）→ 去除
- 内容性提问/评论 → 转为叙事引导句，避免 Q&A 格式

## 叙事性 vs 说明性双轨

- **叙事性内容**（嘉宾亲述的个人经历、关键转折时刻、情绪高峰）→ 使用直接引语，保留冲击力
- **说明性内容**（概念解释、背景、系统原理）→ 精炼书面语准确归纳，受信息增量测试约束

## 极简过渡句

- 话题之间可加 1-2 句过渡，仅用于桥接上下文的跳跃
- 不凭空编造话题间关系

## 专有名词标注

- 人名、公司名、产品名、技术术语、特定概念在首次出现时用中文后括号注明英文原文
- 后续同一名词可只写中文

## 行文风格（强制）

- **bullet point 优先**：主体内容使用 bullet point 短句，每个信息点 2-3 行内讲清
- **段落体限引言/结语**：仅引言和结语允许使用连续段落体，正文均用列表驱动
- **每点独立可扫**：读者扫一眼 bullet 就能抓住信息点，不需通读段落
- **短句原则**：一个 bullet 只承载一个信息维度；如需展开，用子级 bullet 缩进

## 输出格式禁止

- **禁止生成图片/图表/可视化**：全文仅用 markdown 原生格式（文字、表格、列表）
- **表格替代图表**：需要对比数据时用 markdown 对比表格，不生成柱状图/折线图/饼图
- **列表替代流程图**：需要展示流程/层级时用嵌套列表，不生成流程图/思维导图
- **禁止 ASCII art**：不使用字符画、分隔线装饰等非内容元素

## 英文引语翻译

- 原始资料中的英文 Q1 直接引语，必须先准确翻译为中文再引用，不得直接引用英文原文
- 翻译要求：忠实原意、保留原文的口语质地和技术精确性，不添加原文没有的信息
- 翻译后的引语仍标注 Q1，来源标注格式不变

## 多篇特有规则

- 每个话题段落开头必须注明嘉宾姓名，让读者知道谁的发言
- **观点分歧必须呈现**——不为了叙事流畅而让"各方似乎达成共识"。如果原文中观点确实一致，也如实说明"A 和 B 在这一问题上观点一致"
- **不添加原文中没有的"分析""评价""总结"**——文章的价值在于把多篇原文中的共通脉络展示出来，而非代替读者做判断

## 信息来源标注规范

- 每个子话题开头用 `**来源覆盖**：[嘉宾A-文件1]、[嘉宾B-文件2]` 标注信息来源
- 关键数据后用括号标注来源和日期，如"（TME商业化负责人，2026年2月）"
- 避免在文中使用"据采访显示"等模糊表述，应具体到哪位嘉宾、哪份文件

## 主 agent 架构原则

### 章塔尖写作规范

编排模式 5a 中主 agent 起草各章塔尖时，必须遵守：

- **断言句**：章塔尖是一个可验证的判断句，不是一个主题描述。❌ "华为昇腾的产品竞争力" ✅ "华为昇腾在推理市场已构成对英伟达的实质性替代威胁"
- **抽象层级对齐**：所有章塔尖必须在同一抽象层级。不能一章是具体产品指标、另一章是宏观产业趋势
- **MECE**：章节之间互斥（两章不说同一件事）、穷尽（重要维度不遗漏）
- **权重依据**：素材文件数量 × 该话题在原文中的讨论深度 → 决定 1x/1.5x/2x

### 跨章 Q3 注入原则

5c 主 agent 读完所有章节后，注入跨章 Q3：

- 识别两个章中嘉宾观点形成对照的点（同一嘉宾在不同话题中的立场一致性？不同嘉宾对同一件事的隐含分歧？）
- 跨章 Q3 写在相关章节末尾或独立一节，标注 `[跨章 Q3]`
- 不编造章节间本来没有的联系

## Sub-agent 章节写作规范

### 证据分级标注格式

章节中所有观点必须按以下三级标注：

| 等级 | 含义 | 标注格式 |
|------|------|---------|
| **Q1** | 直接引语 / 可验证事实 | 引号包裹原文，后注 `（[嘉宾名]，[文件日期]）`。示例：`"具身智能的落地场景远少于预期"（商汤前高管，20260115-conversation.md）` |
| **Q2** | 归纳转述 | `据[嘉宾名]在[文件]中所述` 开头，基于原文概括。落笔前确认：这句话能否在原文中找到对应段落？ |
| **Q3** | 章内交叉推断 | 标注 `[推理前提：基于 X 和 Y]` + 推理链路。仅在**本章素材范围内**推断，不做跨章判断 |

**跨级禁止**：
- Q3 不得以 Q1 口吻呈现——"嘉宾们一致认为 X"仅在每个嘉宾的原文中可独立验证 X 时才成立
- Q2 不得混入主观延伸——归纳转述限于原文已有内容

### 金字塔写作要求

每章按以下结构组织（主 agent 给定的章塔尖不可修改）：

```
章标题（主 agent 已定）
  ├── 章塔尖展开（1-2 句展开主 agent 的章塔尖）
  ├── 嘉宾 A 的观点 → Q1 引语 + Q2 归纳 + 来源
  ├── 嘉宾 B 的观点 → Q1 引语 + Q2 归纳 + 来源
  ├── ...
  └── 章内交叉分析 → 章级 Q3（标注推理链路，仅限本章素材）
```

塔尖展开原则：
- 先判断是共识还是分歧——不模棱两可
- 共识 → 写出共同认可的具体内容及认可边界
- 分歧 → 写出分歧根因（信息差？立场差？时间差？）

**反模式**：
- ❌ "A 认为 X。B 认为 Y。两人观点不同。"（平铺，无塔尖展开）
- ❌ "A 和 B 在 X 上达成了共识。"（Q3 包装为 Q1）
- ✅ "A 和 B 对 X 的分歧根源在于时间尺度——A 关注 1-2 年短期渗透率（Q1-[fileA]，'2027年前不会超 5%'），B 关注 10 年级别技术替代逻辑（Q1-[fileB]，'最终替代率可达 80%'）。两者核心假设无冲突，因视角尺度不同得出不同结论。"（Q3-[推理前提：两条 Q1 时间尺度不同，核心假设无冲突]）

### 章节格式模板

```markdown
## {章标题}

**来源覆盖**：[嘉宾A-文件X]、[嘉宾B-文件Y]

### 核心判断
[展开主 agent 给定的章塔尖，1-2 句]

### {嘉宾A}的观点
- [Q1] "原文引语..."（[嘉宾名]，[文件日期]）
- [Q2] 据[嘉宾名]所述，...

### {嘉宾B}的观点
（同上结构；3 位以上嘉宾可以合并小节标题为"多方观点"再按嘉宾分段）

### 交叉分析
[章内 Q3，标注推理链路。共识在哪里、分歧在哪里、结构洞在哪里]
```

### 行文与标注规范（强制）

- **bullet point 优先**：主体内容用 bullet point 短句，每个信息点 2-3 行；段落体仅限引言/结语
- **每点独立可扫**：一个 bullet 只承载一个信息维度；需展开用子级 bullet 缩进
- **禁止图片/图表/可视化**：仅用 markdown 原生格式
- **来源标注**：章节开头 `**来源覆盖**：[嘉宾A-文件X]、[嘉宾B-文件Y]`；关键数据括号标注来源+日期
- **观点分歧必须呈现**：不为了叙事流畅硬凑共识。无分歧也如实说明"A 和 B 观点一致"
- **不添加原文没有的评价**

## 全文文章结构模板

适用于标准模式和快速通道的 make-report。编排模式下主 agent 在 5c 拼接时参考此结构。

```markdown
# {大主题}——多篇对谈交叉分析

## 引言
涉及的嘉宾、各自背景、对话发生的上下文。1-3 段。

## 一、{选定子话题 1}
**来源覆盖**：[嘉宾A-文件X]、[嘉宾B-文件Y]

### 核心判断（塔尖）
[一句话概括该话题的核心共识/分歧/元主题]（Q3-[推理前提]）

### {嘉宾A}的观点
[Q1 直接引语 + Q2 归纳转述，附来源标注]

### {嘉宾B}的观点
[同上]

### 交叉分析
[Q3 推断，标注推理链路。共识在哪里、分歧在哪里、结构洞在哪里]

## 二、{选定子话题 2}
（同上结构）

...

## 其他涉及话题
（非选定话题，每话题 1-2 段简要提及，总计不超过全文 20%）

## 结语
（简要提及各篇中未形成交叉的个人亮点，以示对每篇原文的尊重）
```
