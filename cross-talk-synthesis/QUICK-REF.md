# 决策速查 & 陷阱速查

> 主 agent 按需加载本文件（`read` 对应段落），不随 SKILL.md 全量注入上下文。

---

## 决策速查

| 场景 | 行动 | Agent |
|------|------|-------|
| 读取本地 .md 文件 | read 或 task(general) | general |
| 提取结构化信息（≥3 文件/话题）| task(general) 同一条消息并行，自读 EXTRACT-GUIDE.md | general |
| 撰写最终文章（≤2 话题，快速通道）| task(make-report, stream_to_parent=true)，用步骤 2 结果 + read 原文补充 | make-report |
| 撰写最终文章（3 话题，标准模式）| task(make-report, stream_to_parent=true)，用步骤 4 提取结果 | make-report |
| 撰写最终文章（≥4 话题）| 编排模式：5a 架构草案 → 5a-review MECE 审查（条件触发）→ 5a-final 终稿 → 5b 并行章节 → 5c 拼接 | make-report(subtopic) |
| MECE 审查（≥5 章 或 有软性话题）| task(general) × 章节数并行，每个审查一章 | general |
| 需要联网补充检索 | task(gather) | gather（仅限 MCP 工具） |
| 文件 ≤ 10 篇 | 正常流程（步骤 2→6）| — |
| 文件 11-30 篇 | BATCH-STRATEGY：先内容地图后选定 | general |
| 文件 > 30 篇 | BATCH-STRATEGY：强制按子方向分组 | general |

## 陷阱速查表

| 严重度 | 陷阱 | 正确做法 |
|-------|------|---------|
| 🔴 致命 | 把不同时期/上下文的观点硬凑成"共识" | 观点一致也写清各自的时间语境和限定条件 |
| 🔴 致命 | Q3 推断以 Q1 口吻呈现（"嘉宾们一致认为…"）| Q3 必须标注推理链路，区分"嘉宾说"和"我们推断" |
| 🔴 致命 | 过度归纳丢失嘉宾个人语言特征 | Q1 保留原话质地，归纳削长度不削差异性 |
| 🔴 致命 | 编排模式下主 agent 越权写章节正文 | 主 agent 只做架构/拼接/校验，正文由 sub-agent 写 |
| 🔴 致命 | 编排模式 5a 主 agent 读取全部提取文件来定架构 | 主 agent 只读每文件的「塔尖候选」（一行），章节正文素材由 sub-agent 读取 |
| 🟡 重要 | 跳过步骤 3 用户确认 | 必须等用户选定话题后再进入深度提取 |
| 🟡 重要 | 交叉分析平铺"A 说 X，B 说 Y"不做合成 | 先出塔尖核心判断，再看支撑论据，最后交叉分析 |
| 🟡 重要 | 观点未标注来源文件 | 子话题开头标注来源覆盖，关键引语标注文件+日期 |
| 🟡 重要 | ≥4 话题仍用单 agent 写全文 | 编排模式：分章并行写作 + 主 agent 拼接 |
| 🟡 重要 | ≥5 话题时跳过 MECE 审查轮 | 触发 5a-review，并行审查每章 MECE 质量后再定终稿 |
| 🟡 重要 | sub-agent 的章塔尖与主 agent 指定的不一致 | sub-agent 不可修改主 agent 给定的章塔尖，只能展开 |
| 🟢 注意 | 非选定话题占比超过全文 20% | 严格控制在"其他涉及话题"，每话题 1-2 段 |
| 🟢 注意 | 在文中生成图片/图表/可视化 | 仅用 markdown 表格和列表 |
| 🟢 注意 | 长篇段落体影响浏览效率 | bullet point 短句优先，每个信息点 2-3 行 |
| 🟢 注意 | 在步骤 2 就做深度提取 | 轻量扫描只拿最小信息集，详情留给步骤 4 |
| 🟢 注意 | 主 agent prompt 中重复内嵌提取格式 | sub-agent 自读 EXTRACT-GUIDE.md，主 agent 只传文件列表 |
| 🟢 注意 | 快速通道场景仍执行步骤 4 | ≤2 话题 + ≤8 文件 + 无分歧时直接跳步骤 4，步骤 2 结果已够用 |
| 🟢 注意 | sub-agent 做跨章比较或引用其他章素材 | sub-agent 仅限本章素材范围内写作，跨章 Q3 由主 agent 在 5c 完成 |
