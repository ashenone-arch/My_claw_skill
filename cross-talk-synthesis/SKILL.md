# 多篇对谈交叉汇总

> **v2.0 更新**：新增证据分级体系（Q1/Q2/Q3）、强制金字塔输出结构、陷阱速查表；合并冗余速查表、收敛并行警告至执行铁律；移除 PDF 支持（由 pdf-batch-extract 转录为 .md 后使用）；步骤 5 内联文章结构模板；新增 `references/cross-analysis-frameworks.md` 按需加载。

## 概述

将多篇已有对谈文章（不同嘉宾、不同时间、同一主题），按用户选定的话题重新组织为交叉分析文章。

核心设计：**先轻量扫描出内容地图 → 用户选定话题 → 仅对选定话题深度提取和写作**。避免在用户不需要的方向上浪费深度分析。

与 youtube-transcript-to-article 的区别：本 Skill 不涉及字幕下载/VTT 解析等工程步骤，纯「读取已有 .md 文章 → 话题扫描 → 用户选择 → 深度提取 → 交叉写作」。

## 执行铁律

| # | 规则 | 说明 |
|---|------|------|
| 1 | **并行** | ≥3 个文件/话题时，同一条消息中并行发起全部 subagent，严禁串行 |
| 2 | **独立** | 每个文件独立处理，不合并为纯文本文件再处理 |
| 3 | **确认** | 步骤 3（展示内容地图 + 用户选择）不允许跳过 |
| 4 | **溯源** | 每个观点必须标注来源文件路径 + 证据等级（Q1/Q2/Q3） |

## 证据分级

写入和提取时，所有观点按以下三级标注：

| 等级 | 含义 | 提取时要求 | 写作时要求 |
|------|------|-----------|-----------|
| **Q1** | 直接引语 / 可验证事实 | 带引号保留原文 + 来源文件 + 日期 | 用引号呈现，标注 `（[嘉宾名]，[文件日期]）` |
| **Q2** | 归纳转述 | 基于原文压缩概括，标注来源 | 标注 `据[嘉宾]在[文件]中所述` |
| **Q3** | AI 交叉推断 | 跨嘉宾对比/模式识别/元主题 | 标注推理前提 + 推理链路，区分"嘉宾说"和"我们判断" |

**铁律**：Q3 不得以 Q1 口吻呈现。不能说"嘉宾们一致认为 X"，除非每个嘉宾都在对应原文中可验证地表达了 X。

## 金字塔输出结构

**步骤 4 提取和步骤 5 写作均强制使用金字塔**：

```
塔尖（1 句核心判断）
  ├─ 子论点 A → Q1/Q2 锚点（来自哪个文件、哪位嘉宾）
  ├─ 子论点 B → Q1/Q2 锚点
  └─ 交叉分析 → Q3（标注推理链路：前提是什么、推断是什么）
```

**禁止平铺**：不允许 "A 说 X。B 说 Y。两者一致/不一致。" 这种无塔尖判断的流水账。

| 条件 | 行动 |

## 核心流程

### 步骤 1：确定输入文件

- 用户指定目录 → glob 获取所有 `.md` 文件（**仅 .md**，PDF 需先用 pdf-batch-extract 转录）
- 用户列出路径 → 直接使用
- 用户自然语言描述 → 搜索工作区定位

展示文件列表向用户确认。文件 ≤ 10 篇直接进入步骤 2；> 10 篇按 [BATCH-STRATEGY.md](BATCH-STRATEGY.md) 执行。

### 步骤 2：轻量内容扫描（并行）

对每篇文件，只提取最小信息集（不做详细观点摘录）：

- `title`：文件名（去日期和后缀的语义名称）
- `date`：文件日期
- `expert_background`：受访者背景（一句话）
- `topic_tags`：话题标签（3-8 个自然语言短语）
- `one_liner`：一句话摘要（核心内容或最有价值观点）

使用 `task(subagent_type="general")` 并行处理。

> ⚠️ `gather` agent 不能读取本地文件，仅限 MCP 工具。读取本地 .md 必须用 `general` agent。

### 步骤 3：展示内容地图 + 邀请用户选择（必须执行）

基于步骤 2 的扫描结果，按 [BATCH-STRATEGY.md](BATCH-STRATEGY.md) 中的模板展示：

1. **话题频次统计**：哪些话题被最多文件覆盖
2. **文件 × 话题矩阵**：每篇涉及哪些话题
3. **嘉宾背景多样性**：避免只看单一视角

然后**邀请用户选择**：
- 想深度展开的话题（建议 2-5 个）
- 想重点对比的公司（可选）
- 是否排除某些文件或嘉宾

### 步骤 4：针对选定话题深度提取（并行）

仅对步骤 3 用户选定的每个话题，启动 `task(subagent_type="general")` 从相关文件中提取该话题的所有段落。

每个话题的提取必须遵循：

1. **金字塔预判**：先给出该话题的核心判断（塔尖），再逐嘉宾提取论据
2. **证据分级**：每条提取标注 Q1/Q2/Q3
3. **文件溯源**：每条标注来自哪个文件（绝对路径）
4. **分歧呈现**：如有分歧必须标注，如有共识也如实说明——不硬凑共识
5. **尽可能不丢信息**：宁可多提取不要漏

### 步骤 5：分层撰写

按以下要求撰写。**先加载** [WRITING-STANDARDS.md](WRITING-STANDARDS.md) 获取完整写作规范。遇到复杂交叉分析场景时，按需加载 `references/cross-analysis-frameworks.md`。

#### 5.1 文章结构模板

```markdown
# {大主题}——多篇对谈交叉分析

## 引言
涉及的嘉宾、各自背景、对话发生的上下文。1-3 段。

## 一、{选定子话题 1}
**来源覆盖**：[嘉宾A-文件X]、[嘉宾B-文件Y]

### 核心判断（塔尖）
[一句话概括该话题的核心共识/分歧/元主题]（Q3-[推理前提]）

### {嘉宾A}的观点
[Q1 直接引语 + Q2 归纳转述，附来源标注]

### {嘉宾B}的观点
[同上]

### 交叉分析
[Q3 推断，标注推理链路。共识在哪里、分歧在哪里、结构洞在哪里]

## 二、{选定子话题 2}
（同上结构）

...

## 其他涉及话题
（非选定话题，每话题 1-2 段简要提及，总计不超过全文 20%）

## 结语
（简要提及各篇中未形成交叉的个人亮点，以示对每篇原文的尊重）
```

#### 5.2 核心写作要求

- **禁止图表**：全文仅用 markdown 原生格式（文字、表格、列表）。不生成图片、图表、可视化。对比表格替代柱状图/折线图，层级列表替代流程图
- **bullet point 短句**：优先用 bullet point 行使短句，便于快速浏览判断。每个 bullet 2-3 行内讲清一个信息点。段落体仅在引言和结语使用
- **金字塔强制**：每个话题必须先出塔尖判断，再展开
- **证据分级标注**：Q1 用引号，Q2 用归纳表述标注来源，Q3 标注推理链路
- **分歧必须呈现**：不为了叙事流畅而让各方"似乎达成共识"
- **不添加原文没有的分析/评价/总结**——文章的价值在于展示共通脉络
- **选定话题**：深度展开，篇幅与原文讨论深度正相关
- **非选定话题**：归入结尾「其他涉及话题」章节，总计不超过全文 20%

使用 `task(subagent_type="make-report")` 撰写，传入步骤 4 的所有提取结果、WRITING-STANDARDS.md 的写作要求、以及本文的证据分级和金字塔规范。

### 步骤 6：保存文章 + 中间文件

- 最终文章保存至用户指定路径，或默认 `{主题}-多篇对谈交叉汇总.md`
- 步骤 2 的扫描结果和步骤 4 的提取结果保存至 `.alphaclaw/tmp/`，便于后续追溯原文回答问题
- 文件名格式：`.alphaclaw/tmp/{主题}_内容地图.md`、`.alphaclaw/tmp/{话题}_深度提取.md`

## 决策速查

| 场景 | 行动 | Agent |
|------|------|-------|
| 读取本地 .md 文件 | read 或 task(general) | general |
| 提取结构化信息（≥3 文件/话题） | task(general) 同一条消息并行 | general |
| 撰写最终文章 | task(make-report) | make-report |
| 需要联网补充检索 | task(gather) | gather（仅限 MCP 工具） |
| 文件 ≤ 10 篇 | 正常流程（步骤 2→6） | — |
| 文件 11-30 篇 | BATCH-STRATEGY：先内容地图后选定 | general |
| 文件 > 30 篇 | BATCH-STRATEGY：强制按子方向分组 | general |

## 陷阱速查表

| 严重度 | 陷阱 | 正确做法 |
|-------|------|---------|
| 🔴 致命 | 把不同时期/上下文的观点硬凑成"共识" | 观点一致也写清各自的时间语境和限定条件 |
| 🔴 致命 | Q3 推断以 Q1 口吻呈现（"嘉宾们一致认为…"） | Q3 必须标注推理链路，区分"嘉宾说"和"我们推断" |
| 🔴 致命 | 过度归纳丢失嘉宾个人语言特征 | Q1 保留原话质地，归纳削长度不削差异性 |
| 🟡 重要 | 跳过步骤 3 用户确认 | 必须等用户选定话题后再进入深度提取 |
| 🟡 重要 | 交叉分析平铺"A 说 X，B 说 Y"不做合成 | 先出塔尖核心判断，再看支撑论据，最后交叉分析 |
| 🟡 重要 | 观点未标注来源文件 | 子话题开头标注来源覆盖，关键引语标注文件+日期 |
| 🟢 注意 | 非选定话题占比超过全文 20% | 严格控制在"其他涉及话题"，每话题 1-2 段 |
| 🟢 注意 | 在文中生成图片/图表/可视化 | 仅用 markdown 表格和列表；对比表格替代图表，层级列表替代流程图 |
| 🟢 注意 | 长篇段落体影响浏览效率 | bullet point 短句优先，每个信息点 2-3 行；引言和结语例外 |
| 🟢 注意 | 在步骤 2 就做深度提取 | 轻量扫描只拿最小信息集，详情留给步骤 4 |
