# 多篇对谈交叉汇总

## 概述

将多篇已有对谈文章（不同嘉宾、不同时间、同一主题），按用户选定的话题重新组织为交叉分析文章。

核心设计：**先轻量扫描出内容地图 → 用户选定话题 → 仅对选定话题深度提取和写作**。避免在用户不需要的方向上浪费深度分析。

与 youtube-transcript-to-article 的区别：本 Skill 不涉及字幕下载/VTT 解析等工程步骤，纯「读取已有文章 → 话题扫描 → 用户选择 → 深度提取 → 交叉写作」。

## 场景路由

根据输入特征加载对应子文件：

| 条件 | 额外加载 |
|------|---------|
| 文件 > 10 篇 | BATCH-STRATEGY.md（分批策略、内容地图格式） |
| 含 PDF 文件 | PDF-GUIDE.md（pdfplumber 提取方法） |
| 撰写最终文章时 | WRITING-STANDARDS.md（保真底线、标注规范、结构模板） |

## 核心流程

### 步骤 1：确定输入文件

- 用户指定目录 → glob 获取所有 .md/.pdf 文件
- 用户列出路径 → 直接使用
- 用户自然语言描述 → 搜索工作区定位

展示文件列表向用户确认。文件 ≤ 10 篇直接进入步骤 2；> 10 篇按 [BATCH-STRATEGY.md](BATCH-STRATEGY.md) 执行（先分组再扫描）。

> ⚠️ **WARNING**：> 30 篇时必须先按子方向分组，不可全量一次性深度提取。

### 步骤 2：轻量内容扫描（并行）

对每篇文件，只提取最小信息集（不做详细观点摘录）：

- `title`：文件名（去日期和后缀的语义名称）
- `date`：文件日期
- `expert_background`：受访者背景（一句话）
- `topic_tags`：话题标签（3-8 个自然语言短语）
- `one_liner`：一句话摘要（核心内容或最有价值观点）

使用 `task(subagent_type="general")` 并行处理。一般 agent 能读取本地文件。

> ⚠️ **WARNING**
> - `gather` agent 不能读取本地文件，仅限 MCP 工具。读取本地 .md 必须用 `general` agent。
> - ≥ 3 个文件/分组时，必须在**同一条消息中并行**发起全部 subagent，严禁串行。
> - 绝对不要将多个文件合并为一个纯文本文件再处理。每个文件独立提取，便于后续精准定位原文。

### 步骤 3：展示内容地图 + 邀请用户选择（必须执行）

基于步骤 2 的扫描结果，生成「内容地图」展示给用户。格式参见 [BATCH-STRATEGY.md](BATCH-STRATEGY.md) 中的模板：

1. **话题频次统计**：哪些话题被最多文件覆盖
2. **文件 × 话题矩阵**：每篇涉及哪些话题
3. **嘉宾背景多样性**：避免只看单一视角

然后**邀请用户选择**：
- 想深度展开的话题（建议 2-5 个）
- 想重点对比的公司（可选）
- 是否排除某些文件或嘉宾

> ⚠️ **WARNING**：此步骤**不允许跳过**。必须等用户确认后再进入步骤 4。

### 步骤 4：针对选定话题深度提取（并行）

仅对步骤 3 用户选定的每个话题，启动 `task(subagent_type="general")` 从相关文件中提取该话题的所有段落。

每个话题的提取要求：
- 直接引用原文 + 嘉宾姓名 + 关键数据或事实
- 标注「一致性」或「分歧」——观点一致也如实说明
- 标注每条来自哪个文件（绝对路径）
- **尽可能不丢信息**——宁可多提取不要漏

> ⚠️ 所有选定话题在**同一条消息中并行**提取，不逐个串行。

### 步骤 5：分层撰写

加载 [WRITING-STANDARDS.md](WRITING-STANDARDS.md) 中的全部标准，按以下规则撰写：

- **选定话题**：深度展开，篇幅与原文讨论深度正相关。逐公司/逐维度对比，带原文引语和来源标注
- **非选定话题**：归入结尾「其他涉及话题」章节，每话题 1-2 段简要提及，总计不超过全文 20%

使用 `task(subagent_type="make-report")` 撰写，传入步骤 4 的所有提取结果和 WRITING-STANDARDS.md 的写作要求。

### 步骤 6：保存文章 + 中间文件

- 最终文章保存至用户指定路径，或默认 `{主题}-多篇对谈交叉汇总.md`
- 步骤 2 的扫描结果和步骤 4 的提取结果保存至 `.alphaclaw/tmp/`，便于后续追溯原文回答问题
- 文件名格式：`.alphaclaw/tmp/{主题}_内容地图.md`、`.alphaclaw/tmp/{话题}_深度提取.md`

## 决策速查

| 场景 | 行动 | Agent 类型 |
|------|------|-----------|
| 读取本地 .md 文件 | read 或 task(general) | general |
| 提取结构化信息 | task(general) 并行 | general |
| 读取 PDF 文件 | 按 PDF-GUIDE.md 用 pdfplumber | bash(python) |
| 撰写最终文章 | task(make-report) | make-report |
| 单文件/单话题 | 不拆，直接处理 | — |
| ≥ 3 个文件/话题 | 同一条消息并行全部 subagent | general |
| 文件 ≤ 10 篇 | 正常流程（步骤 2-6） | — |
| 文件 11-30 篇 | BATCH-STRATEGY：先内容地图后选定 | general |
| 文件 > 30 篇 | BATCH-STRATEGY：强制按子方向分组 | general |
| 需要联网补充检索 | task(gather) | gather（仅限 MCP 工具） |

## Subagent 类型速查

| Agent | 可用工具 | 适用 | 不适用 |
|-------|---------|------|--------|
| general | 全部工具 | 读取本地文件、提取信息、修改文件 | — |
| gather | MCP 工具（search/query/ask） | 联网检索金融数据库中定性研报 | **不能读本地文件** |
| make-report | read/write/canvas | 撰写分析报告 | 数据检索 |

## 注意事项

- 本 Skill 用于「多篇 → 一篇」汇总。单篇文章引导使用其他工具
- 交叉话题建议控制在 5-10 个——太少粒度太粗，太多信息量不足
- 不强求「交叉」关联——某篇独有的议题不牵强关联，可在引言或结语中单篇提及
- 中间文件必须保留在 `.alphaclaw/tmp/`，便于后续从原始访谈中找到答案
- 用户追问时优先从已有中间文件中查找，不重新检索
