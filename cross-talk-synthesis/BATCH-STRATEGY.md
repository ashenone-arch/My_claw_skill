# 大批量文件处理策略

当文件数量 > 10 篇时，按本策略执行。

## 核心原则

**先快扫出地图，用户指哪打哪。** 不要在用户确认方向前做深度提取。

## 文件数量分级

| 文件数 | 策略 |
|--------|------|
| ≤ 10 篇 | 正常流程：轻量扫描 → 展示话题清单 → 用户选 → 深度提取 |
| 11-30 篇 | 轻量扫描 → 展示「内容地图」→ 用户选择话题和/或公司 → 深度提取选定范围 |
| > 30 篇 | 先展示子方向分组建议 → 用户确认分组 → 按分组分别走轻量扫描 → 用户选择 → 深度提取 |

## 文件分组方法

当文件过多需要分组时，按以下优先级分组：

1. **按文件来源**：高临访谈 vs 内容社区 vs 专题讨论
2. **按子主题**：从文件名中提取共同关键词（如"昆仑芯""竞争格局""推理芯片"）
3. **按时间**：2025 vs 2026

每组 5-8 个文件，不宜过多。每组交给一个 general subagent 做轻量扫描。

## 轻量扫描提取字段

与 ≤ 10 篇不同，大批量时步骤 2 只提取最小信息集：

- `title`：文件名（去除日期和后缀后的语义名称）
- `date`：文件日期
- `expert_background`：受访者背景（一句话）
- `topic_tags`：该访谈覆盖的话题标签（3-8 个自然语言短语）
- `one_liner`：该访谈一句话摘要（核心内容或最有价值的观点）

**不提取** `key_points`（详细观点摘录）——保留到步骤 4 用户选定话题后再做。

## 内容地图格式

扫描完成后，以表格展示给用户：

```markdown
## 内容地图

### 话题频次统计
| 话题 | 覆盖文件数 | 涉及嘉宾 |
|------|-----------|---------|
| 华为昇腾竞争力 | 15 | 商汤前高管、昆仑芯业务主管、科大讯飞工程师... |
| 寒武纪产品评价 | 12 | GPU架构工程师、商汤前高管、销售总监... |
| ...

### 文件 × 话题矩阵
| 文件 | 嘉宾 | 话题A | 话题B | 话题C |
|------|------|-------|-------|-------|
| 文件1 | 嘉宾1 | ✓ | ✓ |  |
| 文件2 | 嘉宾2 | ✓ |  | ✓ |
...

### 嘉宾背景多样性
| 嘉宾类型 | 人数 | 代表性文件 |
|----------|------|-----------|
| 芯片厂商内部人员 | N | ... |
| 下游客户（云厂商/互联网） | N | ... |
| ...
```

## 用户选择

展示内容地图后，邀请用户选择：

1. **想深度展开的话题**（建议 2-5 个）
2. **想重点对比的公司**（可选，如"列出每家公司所有访谈中的 comment 做对比"）
3. **是否排除某些文件或嘉宾**

根据用户选择进入步骤 4（聚焦深度提取）。未选中的话题在最终文章中仅保留简要提及。
