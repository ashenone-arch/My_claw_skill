# 质量检查清单

## MECE 维度交叉检查详细流程

在步骤 4 中，主 agent 必须对 10 句维度塔尖断言执行以下三维检查：

### 维度一：重叠检查

重点检查三对易重叠维度：

**维度 2（周期）↔ 维度 5（逆向）**：
- 通过标准：维度 2 引用周期指标（指数走势、资金流），维度 5 引用行为信号（媒体热度、会议频率）
- 重叠信号：两个维度都引用"市场情绪悲观"作为论据
- 修正方式：维度 2 保留宏观周期判断，维度 5 补充具体的逆向行为信号

**维度 4（下行保护）↔ 维度 10（不对称性）**：
- 通过标准：维度 4 引用最坏情景绝对损失，维度 10 引用情景分析盈亏比
- 重叠信号：两个维度都引用"下行风险有限"
- 修正方式：维度 4 保留压力测试/尾部风险分析，维度 10 补充乐观/悲观收益比率

**维度 3（第二层次思维）↔ 维度 6（信息优势）**：
- 通过标准：维度 3 引用非共识论点与共识的差异，维度 6 引用信息源/分析框架优势
- 重叠信号：两个维度都引用"有深度研究"
- 修正方式：维度 3 保留具体非共识判断，维度 6 补充信息获取渠道或分析方法的独特之处

### 维度二：覆盖检查

- 否决层（2 维）+ 核心层（3 维）+ 增强层（5 维）是否全部有塔尖断言？
- 如果某维度数据不足，检查是否标注了 "[数据不足，已降级处理]"

### 维度三：矛盾检查

- 维度间结论是否自洽？
- 典型矛盾示例：维度一说"低估"（估值底部）但维度二说"市场在贪婪端"（情绪过热）
- 矛盾不是错误，但必须在 MECE 检查中标注，并在步骤 7 风险提示中说明

### MECE 检查报告模板

```
【MECE 检查】
■ 重叠：
  ✅ 无重叠
  / ⚠️ 维度X↔Y 重叠：[具体重叠点] → 已调整边界：[调整方式]
■ 覆盖：
  ✅ 10/10 全覆盖
  / ⚠️ 维度X 数据不足 → [降级处理方案]
■ 矛盾：
  ✅ 无矛盾
  / ⚠️ 维度X↔Y 存在待验证矛盾：[简述] → 已标记，等待步骤7风险提示中说明
```

## 写作前检查清单

步骤 5 启动 make-report 前，主 agent 自问：

| # | 检查项 | 通过标准 |
|---|-------|---------|
| 1 | 数据覆盖 | 3 个 gather 结果文件均已生成且非空？ |
| 2 | 塔尖就绪 | 10 句塔尖断言全部就位？每句标注了证据等级？ |
| 3 | MECE 通过 | 无未解决的重叠/矛盾问题？ |
| 4 | 信息时效 | 定量数据来自最新财报期？定性数据来自近 3 个月？ |
| 5 | 格式合规 | make-report prompt 中已显式禁止生成图片/图表？ |

→ 任一条未通过：先补充，不急于启动 make-report。

## Acceptance Self-check 详细清单

交付前逐项自检：

```
【Acceptance Self-check】

[维度覆盖]
- 10 个维度全部有评估结论？ → ✅/❌
- 每个结论标注证据等级（D1/D2/D3）+ 来源 + 时间？ → ✅/❌
- 证据等级使用正确？（D3 不伪装成 D1）？ → ✅/❌

[否决层]
- 维度一和维度四有明确判断？ → ✅/❌
- 如有"违反框架"是否已直接判定"显著偏离"？ → ✅/❌

[层级权重]
- 增强层上调是否满足"核心层 ≥ 基本符合"的前置条件？ → ✅/❌
- 评级推导路径是否有步步说明？ → ✅/❌

[MECE 一致性]
- 维度 2↔5、4↔10、3↔6 各有独立论证（不引用同一事实）？ → ✅/❌
- 3 个簇的 make-report 输出之间无矛盾？ → ✅/❌

[数据质量]
- 关键结论有 D1/D2 支撑？ → ✅/❌
- D3 结论有推演链路？ → ✅/❌
- 数据缺口三级标注且非空？ → ✅/❌

[格式合规]
- 全文无图片/图表/可视化？ → ✅/❌
- 主体内容 bullet point 驱动？ → ✅/❌

**结论**：✅ 通过 / ❌ 未通过（需补充XXX）
```

**没过 = 没做完，不允许交付。**

## 陷阱速查表（扩展版）

| 严重度 | 陷阱 | 正确做法 | 违反后果 |
|-------|------|---------|---------|
| 🔴 致命 | 否决层"违反框架"但未直接判定"显著偏离" | 任一违反 → 立即停止，判定"显著偏离" | 严重高估标的 |
| 🔴 致命 | D3 主观判断以 D1 口吻呈现 | D3 标注"基于 X+Y → Z" | 评估可信度归零 |
| 🔴 致命 | 数据不足强行下结论 | 标注"[数据不足，已降级处理]" | 幻觉结论 |
| 🟡 重要 | 增强层弥补核心层缺失 | 核心层="部分偏离"→ 不上调 | 虚高评级 |
| 🟡 重要 | 维度 2↔5 引用同一事实 | 独立论证重心 | 重复计分 |
| 🟡 重要 | 组合审查孤立评估 | 额外分析相关性/集中度 | 风险低估 |
| 🟡 重要 | 维度八写成"要保持理性" | 引用具体行为信号 | 无操作价值 |
| 🟡 重要 | make-report 修改塔尖判断 | 只能展开，不可推翻 | 判断与正文脱节 |
| 🟡 重要 | 3 个簇的 make-report 串行执行 | 同一条消息并行发起 | 执行时间翻 3 倍 |
| 🟢 注意 | 信息收集只用 1 个 gather | 3 个并行（快速通道除外） | 数据覆盖不全 |
| 🟢 注意 | 数据缺口留空 | 基于评估结果列出 | 缺口不可见 |
| 🟢 注意 | 簇间术语不统一 | 主 agent 步骤 6 统一 | 可读性下降 |
