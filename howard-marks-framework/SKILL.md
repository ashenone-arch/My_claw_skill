# 霍华德·马克斯投资框架评估

## 执行铁律

| # | 规则 | 说明 |
|---|------|------|
| 1 | **并行** | 深度模式 3 个 gather、3 个 make-report，必须在同一条消息中并行发起 |
| 2 | **溯源** | 每个评估结论标注证据等级（D1/D2/D3）+ 来源 + 数据时间 |
| 3 | **分工** | 主 agent 定塔尖 + 综合判定 + 风险提示；gather 负责检索；make-report 负责维度详细分析（仅深度模式） |
| 4 | **否决优先** | 否决层任一维度"违反框架" → 直接判定"显著偏离" |
| 5 | **纯文本** | 全文只用 markdown 原生格式（文字/表格/列表），禁止生成图片/图表/可视化 |
| 6 | **浅度不写文件** | 浅度模式不调用 write/canvas，不执行 Acceptance Self-check |

## 概述

基于霍华德·马克斯 35 年（1990-2025）完整备忘录提炼的核心投资逻辑，对投资标的或投资组合进行系统化评估。该框架不是选股工具，而是一套"过滤器"——帮助判断一个投资机会是否在正确的价格、正确的时机、具备正确的安全边际。

**双模式架构**：

| 模式 | 数据收集 | 维度分析 | 产物 | 适用场景 |
|------|---------|---------|------|----------|
| 深度 | 3 gather 并行 | 3 make-report 并行（簇 A/B/C） | 完整评估报告 + 保存 .md 文件 | 深度研究、重大决策、投资组合审查 |
| 浅度 | 1 gather | 主 agent 直接输出 | 流式简报（不写文件） | 快速筛选、初步判断、日常速览 |

模式由主 agent 在步骤 1 自动判断（关键词识别），不确定时弹出 AskUserQuestion 让用户二选一。

## 深度模式金字塔架构

```
主 agent 定维度塔尖判断
  ├── MECE 交叉检查
  ├── make-report A：否决层 2 维 —— 并行 ——
  ├── make-report B：核心层 3 维 —— 并行 ——
  ├── make-report C：增强层 5 维 —— 并行 ——
  ├── 主 agent 综合判定
  ├── 风险提示 + 数据缺口
  └── Acceptance Self-check
```

| 层级 | 负责人 | 产物 | 输入 |
|------|--------|------|------|
| 维度塔尖判断（10 句） | 主 agent | 每维一句断言 + MECE 报告 | 3 个 gather 检索结果 |
| 维度详细分析（分 3 簇） | 3 个 make-report 并行 | 3 个簇分析文件 | 对应塔尖判断 + references/dimensions.md |
| 综合判定 + 风险提示 + 数据缺口 | 主 agent | 最终完整评估报告 | 3 个簇分析文件 |

**簇划分**：

| 簇 | 负责维度 | Sub-agent |
|----|---------|-----------|
| A 否决层 | 维度 1（价格）+ 维度 4（风险） | make-report A |
| B 核心层 | 维度 2（周期）+ 维度 3（第二层次）+ 维度 5（逆向） | make-report B |
| C 增强层 | 维度 6-10（信息优势/杠杆/行为心理/时间/不对称性） | make-report C |

## 评估维度速览

十个维度按三层层级组织，第一层具有一票否决权。深度模式维度分析时 make-report 必须自读 `references/dimensions.md` 获取完整评估标准。

| # | 维度 | 层级 | 核心问题 | MECE 边界 |
|---|------|------|---------|----------|
| 1 | 价格与价值 | 否决层 | 价格是否显著低于内在价值？安全边际是否充足？ | — |
| 2 | 市场周期 | 核心层 | 市场处于恐惧端/中间/贪婪端？ | 用周期指标，非行为信号（→维度5） |
| 3 | 第二层次思维 | 核心层 | 判断是否非共识？是否有逻辑支撑？ | 关注思维产出（→维度6 关注投入） |
| 4 | 风险与下行保护 | 否决层 | 最坏情景是否可承受？是否有永久性损失风险？ | 关注绝对损失（→维度10 关注盈亏比） |
| 5 | 逆向思维 | 核心层 | 在市场恐惧时买入、贪婪时回避？ | 用行为信号（→维度2 用周期指标） |
| 6 | 信息优势 | 增强层 | 在低效市场是否有可论证的优势？ | 关注思维投入（→维度3 关注产出） |
| 7 | 杠杆与资本结构 | 增强层 | 杠杆是否可控？最坏情景能否存活？ | — |
| 8 | 行为心理学 | 增强层 | 决策是否受情绪驱动？能否在逆境坚持？ | — |
| 9 | 时间与耐心 | 增强层 | 是否有足够时间让投资逻辑验证？ | — |
| 10 | 不对称性 | 增强层 | 上行空间是否显著大于下行风险？ | 关注盈亏比（→维度4 关注绝对损失） |

> 完整评估标准（核心原则、评估要点、判断标准）见 `references/dimensions.md`。

## 证据等级体系

所有评估结论必须标注证据等级。没有证据等级 = 不能写入报告。

| 标签 | 含义 | 示例 |
|------|------|------|
| D1 | 可验证的客观数据 | 财报数字、行情数据、估值指标、杠杆比率 |
| D2 | 可追溯的市场信号 | 分析师观点、资金流向、新闻覆盖频次 |
| D3 | 基于框架的主观判断 | 周期位置判断、情绪判断、非共识推演 |

**约束**：关键结论必须有 D1/D2 支撑。D3 结论必须附推演链路（"基于 X + Y → Z"）。禁止 D3 以 D1 口吻呈现。

## 时间精度规则

| 用户表述 | 查询精度 | date_range |
|---------|---------|-----------|
| "今天/盘后/盘中/这两天" | 日级别 | `past_day` |
| "本周/这周/过去一周" | 周级别 | `past_week` |
| "本月/这个月/近一个月" | 月级别 | `past_month` |
| "最近/近期/最新"/"本季度/近三个月" | 季度级别 | `past_quarter`（默认） |
| "近半年" | 半年级别 | `past_half_year` |
| "近一年/去年" | 年级别 | `past_year` |

**原则**：用户说了具体时间单位 → 必须用对应精度，禁止扩大范围。

## 模式判断（步骤 1 执行）

主 agent 在步骤 1 识别评估对象的同时，扫描用户原话中的模式关键词：

| 关键词 | → 模式 |
|--------|--------|
| "深度/完整/详细/全面/出报告/写报告/保存" | 深度模式 |
| "快速/简要/速览/简单/浅度/简报/粗略/看一眼" | 浅度模式 |

无明确关键词时，主 agent 内部判断：
- 投资组合（≥ 2 个标的）→ 默认深度模式
- 单标的 + 问题简短（如"XX 符合马克斯框架吗"）→ 默认浅度模式
- 无法判断 → 弹出 AskUserQuestion 让用户二选一

> 确认模式后，按对应模式执行后续步骤，不再回退。

## 数据源分工

| 数据类型 | 工具 | 原则 |
|---------|------|------|
| 结构化（财务/估值/行情/风险） | iFinD MCP（`mcporter call`）+ `query_finance_data` | 定量数据不从研报里找数字 |
| 定性（研报观点/纪要/策略） | AlphaClaw `search_finance_reports` | 定性判断不用数据库替代 |
| 实时（新闻/政策/热度） | `websearch` + iFinD news MCP | 实时信号不用滞后的研报 |

> 详细映射（每个维度的 P0/P1/P2 数据源 + 具体查询模板）见 `references/data-source-mapping.md`。

## 执行步骤

### 步骤 1：识别评估对象 + 模式判断

确认用户要评估的是什么（单标的/投资组合），提取标的名称和代码。同时按「模式判断」规则确定深度/浅度模式。组合模式默认深度。

## 深度模式

> **触发条件**：用户明确要求深度/完整/详细分析，或为投资组合审查，或关键词判断 + 用户确认。

### 深度-步骤 1：并行数据收集

> 执行前读取 `references/data-source-mapping.md` 获取 gather prompt 模板。

在同一条消息中并行发起 3 个 `task(subagent_type="gather")`：

- **gather A — 定量数据**：财务指标、估值、行情、杠杆比率、情景数据
- **gather B — 定性分析**：研报观点、分析师纪要、风险讨论、行业策略
- **gather C — 实时动态**：近期新闻、行业趋势、市场情绪信号、政策变化

每个 gather 的 prompt 使用 `references/data-source-mapping.md` 中的模板，传入用户原始问题 + 当前日期。必须设置 `save_to_file=True` 和 `file_path`。

> **组合模式**：每个标的 × 3 gather 并行，额外 1 个组合层面 gather。

### 深度-步骤 2：主 agent 预判 — 维度塔尖判断 + MECE 检查

读取 gather 结果文件，基于「评估维度速览」表对 10 个维度各写一句断言（塔尖判断），标注证据等级 + 来源。

**塔尖写作规范**：断言句，不用主题描述句。❌ "价格与价值维度分析" ✅ "当前 PE 12x，低于 5 年均值 18x，处于历史底部 15% 分位，存在安全边际"。数据不足时标注 `[数据不足，已降级处理]`。

然后执行 MECE 交叉检查（详细流程见 `references/quality-checks.md`）：

```
【MECE 检查】
■ 重叠：✅ 无重叠 / ⚠️ 维度X↔Y 重叠，已调整边界
■ 覆盖：✅ 10/10 全覆盖 / ⚠️ 维度X 数据不足
■ 矛盾：✅ 无矛盾 / ⚠️ 维度X↔Y 待验证矛盾
```

向用户展示塔尖判断 + MECE 结果，确认后进入深度-步骤 3。

### 深度-步骤 3：并行维度分析（3 个 make-report）

> 写入前读取 `references/quality-checks.md`「写作前检查清单」节，5 条全部通过后再启动。
> 每个 make-report 自读 `references/dimensions.md` 获取对应维度的完整评估标准（核心原则、评估要点、判断标准、MECE 边界）。

在同一条消息中并行发起 3 个 `task(subagent_type="make-report", stream_to_parent=true)`：

| Sub-agent | 负责维度 | prompt 内容 |
|-----------|---------|-------------|
| make-report A | 维度 1 + 维度 4 | 簇 A 简报：两维度塔尖 + 检索文件路径 |
| make-report B | 维度 2 + 维度 3 + 维度 5 | 簇 B 简报：三维度塔尖 + MECE 边界 + 检索文件路径 |
| make-report C | 维度 6-10 | 簇 C 简报：五维度塔尖 + MECE 边界 + 检索文件路径 |

**簇简报格式**（以簇 A 为例）：

```
撰写「否决层维度分析」章节。自读 references/dimensions.md 获取维度 1 和维度 4 的完整评估标准。

维度一塔尖：[断言句]
维度四塔尖：[断言句]

数据素材：{检索文件路径列表}

格式要求：
- 每个维度输出：塔尖判断展开（150-200 词）+ 评估结论（符合/警惕/违反）+ 核心论据（2-3 句）+ 数据锚点（标注 D1/D2/D3 + 时间）
- 只展开论证塔尖判断，不可推翻
- 数据与塔尖矛盾时标注 "[数据与预判矛盾]"
- 禁止生成图片/图表/可视化，全文仅 markdown 原生格式
```

### 深度-步骤 4：主 agent 综合判定

读取 3 个 make-report 输出文件，提取各维度评级，执行三层级权重计算：

**第一层：否决层**

| 维度 | 否决理由 |
|------|---------|
| 价格与价值 | 价格显著高于内在价值 |
| 风险与下行保护 | 最坏情景不可承受或永久性资本损失 |

- 任一否决维度 = "违反框架" → 直接判定 **显著偏离**
- 两者同时"需要警惕" → 最高评级降为 **部分偏离**

**第二层：核心层**（市场周期 / 第二层次思维 / 逆向思维）

- 3 个全部"符合" → 上限 **强符合**
- 2 个"符合" → 上限 **基本符合**
- 0-1 个"符合" → 上限 **部分偏离**

**第三层：增强层**

| 条件 | 调整 |
|------|------|
| ≥ 4 个"符合" **且** 核心层 ≥ "基本符合" | 上调一档 |
| ≥ 3 个"警惕/违反" | 下调一档 |
| 核心层 = "部分偏离" 且 ≥ 4 个"符合" | **不调整** |

**整体评级**：强符合 / 基本符合 / 部分偏离 / 显著偏离。写出评级推导路径（否决层 → 核心层上限 → 增强层微调 → 最终评级）。

### 深度-步骤 5：风险提示 + 数据缺口 + 交付

1. **风险提示**：列出不确定性；引用马克斯核心告诫；标注 MECE 中发现的特验证矛盾
2. **数据缺口**（三级）：
   - **必需**（影响核心判断）：{缺失项} → 应从 {数据源} 获取
   - **建议**（提升评估质量）：{缺失项}
   - **不确定但影响判断**：{假设性问题}
3. 组装完整报告，按 `references/output-format.md` 的模板结构
4. 保存并预览：
   - 单标的：`D:\AlphaClaw\Output\马克斯框架评估\{标的名称}-{代码}\{yyyymmdd}-评估.md`
   - 投资组合：`D:\AlphaClaw\Output\马克斯框架评估\组合-{组合名称}\{yyyymmdd}-审查.md`

### 深度-步骤 6：Acceptance Self-check

按 `references/quality-checks.md`「Acceptance Self-check 详细清单」节逐项执行。没过 = 没做完，不允许交付。

## 浅度模式

> **触发条件**：用户明确要求快速/简要/速览，或单标的 + 问题简短，或关键词判断 + 用户确认。

### 浅度-步骤 1：轻量数据收集

> 执行前读取 `references/data-source-mapping.md` 获取 gather prompt 模板。

发起 **1 个** `task(subagent_type="gather")`，全维度检索（定量 + 定性 + 实时合并），`recall_num=12`。必须设置 `save_to_file=True` 和 `file_path`。

### 浅度-步骤 2：塔尖判断 + 简报输出（主 agent 直接完成）

读取 gather 结果文件，基于「评估维度速览」表对 10 个维度各写一句断言（塔尖判断），标注证据等级 + 来源。塔尖写作规范同深度模式。

然后直接输出简报，包含以下结构：

1. **10 维塔尖判断**（每维一句断言 + 评级：符合/警惕/违反 + 证据等级 D1/D2/D3）
2. **综合判定**：按三层级权重法快速评级（否决优先 → 核心层 → 增强层微调）
3. **风险提示**：1-3 句关键不确定性
4. **数据缺口**（如有明显缺失）

**浅度模式铁律**：
- 不调用 write/canvas，不保存文件
- 不调用 make-report sub-agent
- 不执行 MECE 检查、Acceptance Self-check
- 每维 80-120 字（塔尖断言 + 评级 + 证据），不做详细展开
- 否决优先原则照常生效

**浅度简报模板**：

```
## {标的名称} 马克斯框架速览

### 10 维塔尖

| # | 维度 | 评级 | 断言 |
|---|------|------|------|
| 1 | 价格与价值 | 符合/警惕/违反 | {一句断言} [D1/D2/D3, 时间] |
| ... | （其余 9 维同理） |

### 综合判定：强符合/基本符合/部分偏离/显著偏离

推导路径：否决层 {通过/触发} → 核心层 {X/3 符合} → 增强层 {上调/下调/不变} → 最终评级

### 风险提示
- {关键风险 1}
- {关键风险 2}

### 数据缺口
- {如有明显缺失}
```

## 陷阱速查表

| 严重度 | 陷阱 | 正确做法 |
|-------|------|---------|
| 🔴 致命 | 否决层"违反"但未判定"显著偏离" | 立即停止，直接判定 |
| 🔴 致命 | D3 以 D1 口吻呈现 | D3 附推演链路 |
| 🔴 致命 | 数据不足强行下结论 | 标注 `[数据不足]` |
| 🟡 重要 | 增强层弥补核心层缺失 | 核心层="部分偏离"→ 不上调 |
| 🟡 重要 | 维度 2↔5 引用同一事实 | 各自独立论证重心 |
| 🟡 重要 | 组合审查孤立评估 | 额外分析相关性 |
| 🟡 重要 | make-report 修改塔尖判断 | 只能展开，不可推翻 |
| 🟡 重要 | 3 个 make-report 串行执行 | 同一条消息并行发起 |
| 🟡 重要 | 浅度模式调用 write/canvas | 浅度模式不写文件，流式输出即可 |
| 🟢 注意 | 深度模式只用 1 个 gather | 深度模式 3 个并行，浅度模式用 1 个 |
| 🟢 注意 | 数据缺口留空 | 基于评估结果列出 |
| 🟢 注意 | 簇间术语不统一 | 主 agent 深度-步骤 5 统一 |
| 🟢 注意 | 浅度模式调用 make-report | 浅度模式主 agent 直接输出，不委托 |
| 🟢 注意 | 浅度模式执行流程过长 | 浅度模式仅 2 步：gather → 简报 |

> 扩展版陷阱说明见 `references/quality-checks.md`

## 注意事项

1. **这是评估框架，不是选股公式**：用于判断一个投资机会是否"符合马克斯的投资思维模式"
2. **重视定性判断**：投资者心理、市场情绪、周期位置的判断与定量分析同样重要
3. **避免教条主义**：马克斯强调"确定性在投资中没有位置"，所有结论都带有不确定性
4. **价格高于质量**：一家"伟大公司"在错误价格下可能是糟糕的投资——否决层存在的根本原因
5. **优先评估下行风险**：先看"哪里可能出错"，再看"哪里可能做对"
6. **保持智识谦逊**：信息不足时在数据缺口中明确标注，不强行下结论
7. **MECE 边界不能忘**：维度 2↔5、4↔10、3↔6 各有独立论证重心
8. **浅度模式不等于低质量**：简报虽然精简，但证据等级、否决优先、三层级权重等核心原则照常执行
