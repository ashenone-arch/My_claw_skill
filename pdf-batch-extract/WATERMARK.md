# 水印清理专项指引

当检测到 PDF 带有水印（如"严格保密，仅供内部参考"等竖排侧栏文字）时，按以下流程处理。

## 识别水印

扫描 PDF 前 3 页的文本，识别以下特征：
- 逐字分行散布的单字符（如「严」「密」「仅」「云」「基」等独立成行）
- 重复出现的短行（如「参」「部」「内」「金」「基」「锋」「云」「供」「仅」单字成行）
- 竖排 Confidentiality 风格水印混在正文行间

**自动识别逻辑**：统计每页前 3 行和后 3 行中出现频率高的单字，若属于"严格保密仅供云锋基金内部参考"等水印字符集，则判定有水印。

## 询问水印内容

发现水印后，主动询问用户：

> "检测到 PDF 带有水印，请提供水印文字（如'严格保密，仅供云锋基金内部参考'），以便后续精准清理。"

若用户未提供水印文字，跳过清理步骤（不阻断提取流程）。

## 水印清理算法

### 字符分类

将水印字分为两类（基于竖排侧栏位置奇偶性）：

| 类别 | 字符 | 说明 |
|------|------|------|
| **必须清理**（奇数位） | 严、密、保、仅、云、基、参 | 水印侧栏字，无论出现在哪里均视为噪声 |
| **谨慎清理**（偶数位） | 格、密、供、锋、金、部、考 | 同时也是正常汉字，需结合上下文判断 |

### 清理规则

#### 规则 1：行首水印字（仅奇数位）→ 跳过
```
输入: "严各位来宾，同仁大家好"
输出: "各位来宾，同仁大家好"
```
仅删除行首的奇数位水印字，偶数位字保留（可能是正文字）。

#### 规则 2：行尾水印字（仅奇数位）→ 截断
```
输入: "谢谢大家严"
输出: "谢谢大家"
```
仅删除行尾的奇数位水印字，偶数位字但前字是中文/英文/数字时也截断。

#### 规则 3：词间水印字（奇数位）→ 移除
```
输入: "智能体础设施"
输出: "智能体基础设施"
```
前后都是中文时，判定为词间水印字，直接移除。

#### 规则 4：词间偶数位水印字 → 高置信场景修复
对于确认为水印破坏的词，给出精确替换：

| 水印破坏 | 修复为 |
|---------|--------|
| 础设施 | 基础设施 |
| 普惠础 | 普惠基础 |
| 核心建 | 核心基建 |
| 供链 | 供应链 |
| 应链 | 供应链 |
| 部现流 | 现金流 |
| 国首家/国首款/国前列 | 国内首家/国内首款/国内前列 |
| 保管理层 | 管理层 |
| 署到运维 | 部署到运维 |
| 云GPU | 保持不变（云是奇数位但出现在正确位置） |

#### 规则 5：纯水印行 → 跳过
整行仅包含水印字（可能带逗号/顿号），直接跳过。

#### 规则 6：孤立单字行 → 跳过
仅 1-2 个水印字单独成行（如 `严`、`供`），直接跳过。

### 特殊说明

- **保留合法词语**：水印字也是正常汉字（如「基」「金」「供」「部」「考」等），在合法词语（如「基建」「资金」「供给」「部门」「成功」）中出现的不做处理
- **竖排水印残留**：竖排 Confidentiality 水印在 pdfplumber 提取后可能残留少量单字碎片（如行尾挂「严」），通过规则 1/2 处理
- **不要 OCR**：本指引专注规则引擎清理，不处理扫描件 PDF（需要 OCR 那是另一个 workflow）

## 清理脚本用法

在提取脚本传入 `wm_chars` 参数：

```python
from scripts.extract_pdf_to_md import batch_extract

# 无水印
batch_extract(pdf_dir)

# 有水印时，传入需清理的水印字符集（奇数位全部 + 偶数位中确实混入的）
wm_odd  = set('严保密仅云基参')   # 必须清理
wm_even = set('格密供锋金部考')   # 谨慎清理
all_wm = wm_odd | wm_even

batch_extract(pdf_dir, wm_chars=all_wm)
```

提取脚本内部会将 `wm_chars` 传给 `extract_pdf()` 中的 `clean_text_block()` 函数，按规则 1-6 自动清理。精确词间修复（如「础设施」→「基础设施」）由 subagent 在提取后额外执行。
