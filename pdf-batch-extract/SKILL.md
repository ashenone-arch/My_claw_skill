# PDF 批量原文提取（含表格）

## 概述

将指定文件夹内的所有 PDF 文件逐一读取，同时提取全文文本和表格，保存为与之对应的同名 .md 文件。文本自动清理页眉/页脚/页码，表格以 Markdown 表格格式嵌入对应页面位置。不进行话题分析、摘要生成或交叉对比。

## 使用场景

- 用户指定一个文件夹，希望把里面所有 PDF 批量转成 markdown
- 用户上传了一批 PDF，想快速获得每份 PDF 的完整内容（文本 + 表格）
- PDF 中包含大量表格（财报表格、数据表、统计表等），需要保留表格结构
- 后续需要对文本做进一步分析，但当前只需完成 PDF→MD 这一步
- 用户说"帮我把这个文件夹里的 PDF 都转成 md，表格也要保留"

## 执行步骤

### 步骤 1：确定目标文件夹

用户可能通过以下方式指定：

- 直接给出文件夹路径（如 `D:\AlphaClaw\reports`）
- 说"这个文件夹"（需要确认当前工作目录下的某个子文件夹）
- 只说"处理这个文件夹"，但未指明——向用户确认具体路径

扫描目标文件夹，获取所有 `.pdf` 文件列表（不区分大小写），按文件名排序。

向用户展示扫描结果，格式如下：

```
找到 N 个 PDF 文件：

1. 文件名A.pdf（页面数：X）
2. 文件名B.pdf（页面数：Y）
3. ...

将提取到同目录下的 .md 文件中。
```

如果 PDF 数量过多（> 20 个），建议用户分批处理或筛选。

### 步骤 2：检查并安装 pdfplumber

```bash
python -c "import pdfplumber; print('ok')"
```

如果报错 `ModuleNotFoundError`，先安装：

```bash
python -m pip install pdfplumber --quiet
```

安装成功后才进入步骤 3。

### 步骤 3：执行提取

**脚本位置**：`{技能目录}/scripts/extract_pdf_to_md.py`

提取脚本负责：
- 文本提取 + 清洗（页眉/页脚/页码自动检测并移除）
- 表格提取 + Markdown 格式化（`page.extract_tables()` → Markdown table）
- 分批读取防超时（> 20 页按 10 页/批）
- 汇总 JSON 输出（`_pdf_extract_summary.json`）

脚本支持两种模式：
- **批量模式**：`python "{技能目录}/scripts/extract_pdf_to_md.py" "{目标文件夹绝对路径}"`
- **单文件模式**：`python "{技能目录}/scripts/extract_pdf_to_md.py" --single "{PDF绝对路径}" "{输出目录}"`

根据 PDF 数量选择策略：

**策略 A：PDF 数量 ≤ 2，直接执行脚本**

```bash
python "{技能目录}/scripts/extract_pdf_to_md.py" "{目标文件夹绝对路径}"
```

**策略 B：PDF 数量 ≥ 3，通过 task 工具并行启动 subagent**

每个 subagent 负责 1 个 PDF。在同一条消息中并行发出所有 task 调用。

每个 subagent 的 prompt：

```
你负责读取以下 PDF 并输出对应的 markdown 文件。

请用 pdfplumber 执行提取脚本，完成以下操作：

1. 读取 PDF，统计总页数
2. 对全 PDF 做页眉/页脚识别：统计所有页面首行和末行，出现次数 ≥2 的行视为页眉/页脚
3. 对每页执行：
   a. 提取文本 `page.extract_text()`，清理页眉/页脚/页码
   b. 提取表格 `page.extract_tables()`，将每个非空表格转为 Markdown 表格格式
   c. 文本先行、表格追加，用 `--- 第 X 页 ---` 分隔页
4. PDF 超过 20 页时分批读取（每批 10 页），复用步骤 2 的页眉/页脚识别结果
5. 输出同名 .md 文件，格式如下：

---
# {文件名（不含后缀）}

> 来源：{原文件名}  |  共 {N} 页

---

--- 第 1 页 ---
{清理后的文本}

**表 1-1：**

| 列1 | 列2 | 列3 |
| --- | --- | --- |
| ... | ... | ... |

--- 第 2 页 ---
...

---

6. 文件保存到输出目录，编码 UTF-8
7. Markdown 表格规则：
   - 第一行是列标题，第二行是 `| --- | --- | ... |`
   - 清洗单元格内容（去除首尾空白、换行替换为空格）
   - 跳过全空行和全空表格
   - 表格编号格式：`表 {页码}-{序号}`

完成后汇报：文件名、页数、md 文件路径。

具体参数：
- PDF 路径：{pdf_absolute_path}
- 输出目录：{output_dir}
```

### 步骤 4：展示汇总结果

全部完成后，向用户汇报：

```
✅ 提取完成！

| 序号 | PDF 文件 | 状态 | 页数 | 表格数 | MD 文件 |
|------|---------|------|------|--------|---------|
| 1 | 文件A.pdf | 成功 | 15 | 3 | 文件A.md |
| 2 | 文件B.pdf | 成功 | 32 | 8 | 文件B.md |
| ... | ... | ... | ... | ... | ... |

汇总文件：_pdf_extract_summary.json
输出目录：{文件夹路径}
```

如有失败项，单独标注并说明原因（密码保护 PDF、扫描件无文字等）。

## 输出格式

每个 .md 文件格式：

```markdown
# {文件名（不含 .pdf 后缀）}

> 来源：{原文件名}  |  共 {N} 页

---

--- 第 1 页 ---
{清理后的文本段落}

**表 1-1：**

| 列A | 列B | 列C |
| --- | --- | --- |
| 数据1 | 数据2 | 数据3 |

--- 第 2 页 ---
{清理后的文本段落}

**表 2-1：**

| 列X | 列Y |
| --- | --- |
| ... | ... |

...（按实际页数依次列出）
```

**表格格式化规则**：
- 第一行始终作为表头（`| --- | --- |` 分隔）
- 单元格内换行替换为空格，保持单行
- 空行自动跳过
- 编号格式 `{页码}-{该页表格序号}`，方便定位原文

## 注意事项

- **扫描而非猜测**：用户没有指定文件夹时，不要假设路径，先向用户确认
- **同文件跳过**：如果同目录下已有同名 .md 文件，提示用户是否覆盖，不自动覆盖
- **失败不阻断**：某个 PDF 失败不影响其他 PDF 的处理，所有结果汇总展示
- **分页防超时**：超过 20 页的 PDF 必须分批读取，每批 10 页，避免单次超时
- **分页边界**：分批读取时用 `pdf.pages[start:end]` 切片，不重复读取整本 PDF
- **中文路径**：Python 3.13+ 原生支持中文路径，传入绝对路径字符串即可
- **不要合并**：所有 PDF 各自独立输出为独立的 .md 文件
- **不要分析**：纯原文提取，不做摘要、话题识别或结构化提取（那是 cross-talk-synthesis 的工作）
- **页眉/页脚/页码自动清理**：`detect_headers_footers()` 统计识别 + 正则兜底，无需用户指定
- **表格非强制**：PDF 无表格时正常输出纯文本，不影响提取流程
- **表格不跨页合并**：每页表格独立提取编号，跨页表格暂不自动拼接（避免误合并不同表格）
- **文本表格并存**：同一页中，表格内容会以两种形式出现——`extract_text()` 提取的原始文本行（保留阅读流），以及 `extract_tables()` 生成的 Markdown 表格（保留数据结构）。这是 pdfplumber 的正常行为，不做去重处理（原始文本行可在表格格式异常时作为兜底参考）
