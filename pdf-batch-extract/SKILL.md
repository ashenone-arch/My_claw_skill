# PDF 批量原文提取（含表格）

## 概述

将指定文件夹内的所有 PDF 文件逐一读取，同时提取全文文本和表格，保存为同名 .md 文件。文本自动清理页眉/页脚/页码，表格以 Markdown 表格格式嵌入对应页面位置。不进行话题分析、摘要生成或交叉对比。

> **辅助文件**：
> - `scripts/extract_pdf_to_md.py` — 提取核心脚本
> - `WATERMARK.md` — 水印检测与清理专项指引
> - `QUICKREF.md` — 快速参考表（场景决策、常见问题）

## 使用场景

- 用户指定一个文件夹，希望把里面所有 PDF 批量转成 markdown
- 用户上传了一批 PDF，想快速获得每份 PDF 的完整内容（文本 + 表格）
- PDF 中包含大量表格（财报表格、数据表、统计表等），需要保留表格结构
- 后续需要对文本做进一步分析，但当前只需完成 PDF→MD 这一步

## 执行步骤

### 步骤 0：环境预检

**先检查环境，再开始**：

```bash
python -c "import pdfplumber; print('ok')"
```

- 输出 `ok` → 环境完备，跳到步骤 1
- 报错 `ModuleNotFoundError` → 执行 `python -m pip install pdfplumber --quiet`，安装成功后再继续

### 步骤 1：确定目标文件夹

扫描目标文件夹，获取所有 `.pdf` 文件列表（不区分大小写），按文件名排序。

向用户展示扫描结果：

```
找到 N 个 PDF 文件：

1. 文件名A.pdf（页面数：X）
2. 文件名B.pdf（页面数：Y）
3. ...

将提取到同目录下的 .md 文件中。
```

> 如果 PDF 数量 > 20，建议用户分批处理或筛选。

### 步骤 1.5：水印检测（自动进行，发现后询问）

读取每个 PDF 的前 3 页文本，检查是否有水印特征：
- 逐字分行散布的单字符（如「严」「密」「仅」「云」「基」等）
- 重复出现的纯水印单字行（如「参」「部」「内」「金」「基」「锋」「云」「供」「仅」）
- 竖排 Confidentiality 风格水印混入正文行间

**如果检测到水印**，主动询问用户：

> "检测到 PDF 带有水印，请提供水印文字内容（如'严格保密，仅供内部参考'），以便后续精准清理。如果水印内容复杂或不确定，可直接说'跳过'，我会提取后用规则清理。"

- 用户提供水印文字 → 记录水印内容，跳到步骤 2（按 WATERMARK.md 清理）
- 用户说"跳过" → 不清理，继续步骤 2（不阻断流程）

### 步骤 2：执行提取

**脚本路径**：`{技能目录}/scripts/extract_pdf_to_md.py`

根据 PDF 数量选择策略：

**PDF 数量 ≤ 2**：直接执行脚本

```bash
python "{技能目录}/scripts/extract_pdf_to_md.py" "{目标文件夹绝对路径}"
```

**PDF 数量 ≥ 3**：并行启动 subagent，每个 subagent 处理 1 个 PDF。

在同一消息中并行发出所有 task 调用（不要串行）。每个 subagent 的 prompt：

```
你负责读取以下 PDF 并输出对应的 markdown 文件。

执行步骤：
1. 用 pdfplumber.open() 读取 PDF，统计总页数
2. 调用 detect_headers_footers() 识别页眉/页脚（出现次数 >= 阈值的首行/末行）
3. 对每页执行：
   a. page.extract_text() → 清理页眉/页脚/页码（跳过识别到的页眉/页脚行）
   b. page.extract_tables() → 非空表格转为 Markdown（表 1-1 格式）
   c. 文本先行、表格追加，--- 第 X 页 --- 分隔页
4. 超过 20 页时用 pdf.pages[start:end] 分批读取（每批 10 页），复用步骤 2 的页眉/页脚识别结果
5. 有水印时（第 1.5 步已告知），按 WATERMARK.md 中的规则清理：
   - 行首/行尾的奇数位水印字跳过
   - 词间奇数位水印字移除
   - 精确词间修复（础设施→基础设施，供链→供应链 等）
6. 输出 MD 文件，UTF-8 编码，格式：

---
# {文件名（不含后缀）}

> 来源：{原文件名}  |  共 {N} 页

---

--- 第 1 页 ---
{清理后的文本}

**表 1-1：**
| 列1 | 列2 | 列3 |
| --- | --- | --- |
| ... | ... | ... |

--- 第 2 页 ---
{清理后的文本}
...

具体参数：
- PDF 路径：{pdf_absolute_path}
- 输出目录：{output_dir}

完成后汇报：文件名、页数、表格数、md 文件路径。
```

### 步骤 3：展示汇总结果

```
✅ 提取完成！

| 序号 | PDF 文件 | 状态 | 页数 | 表格数 | MD 文件 |
|------|---------|------|------|--------|---------|
| 1 | 文件A.pdf | 成功 | 15 | 3 | 文件A.md |
| 2 | 文件B.pdf | 失败 | - | - | - |

汇总文件：_pdf_extract_summary.json
输出目录：{文件夹路径}
```

失败项单独标注原因（密码保护 PDF、扫描件无文字等）。

## 输出格式

每个 .md 文件格式：

```markdown
# {文件名（不含 .pdf 后缀）}

> 来源：{原文件名}  |  共 {N} 页

---

--- 第 1 页 ---
{清理后的文本段落}

**表 1-1：**
| 列A | 列B | 列C |
| --- | --- | --- |
| 数据1 | 数据2 | 数据3 |

--- 第 2 页 ---
...
```

**表格格式化规则**：
- 第一行作为表头，第二行 `| --- | --- |` 分隔
- 单元格内换行替换为空格，保持单行
- 空行/全空表格跳过
- 编号格式：`表 {页码}-{该页表格序号}`

## 注意事项

### ⚠️ TRAPS（易错陷阱）

- **不重复安装**：步骤 0 已检查环境，任何情况下不要重复执行 `pip install pdfplumber`
- **不猜测路径**：用户没有明确指定文件夹时，先询问，不要假设路径
- **不覆盖已有文件**：同目录下已有同名 .md 时，必须询问用户是否覆盖
- **分批防超时**：超过 20 页必须分批读取，每批 10 页，页眉/页脚识别结果复用全局
- **水印先问再清**：步骤 1.5 检测到水印后，必须先询问用户再清理，不自行猜测水印内容
- **不合并**：所有 PDF 各自独立输出为独立 .md，不合并为一个文件
- **不分析**：纯原文提取，不做摘要、话题识别、结构化提取（那是 cross-talk-synthesis 的工作）

### ℹ️ 技术说明

- **中文路径**：Python 3.13+ 原生支持中文路径，传入绝对路径字符串即可
- **表格不跨页合并**：每页表格独立编号，跨页表格暂不自动拼接
- **文本表格并存**：同一页中 `extract_text()` 和 `extract_tables()` 都会输出表格内容，这是 pdfplumber 正常行为，不做去重（原始文本行可在表格格式异常时作为兜底参考）
- **扫描件 PDF**：无文字层（全是图片）的 PDF，pdfplumber 返回空文本，跳过并标记为失败
