# PDF 批量原文提取（含表格）

## 概述

将指定文件夹内的所有 PDF 文件逐一读取，同时提取全文文本和表格，保存为同名 .md 文件。文本自动清理页眉/页脚/页码，表格以 Markdown 表格格式嵌入对应页面位置。不进行话题分析、摘要生成或交叉对比。

> **辅助文件**：
> - `scripts/extract_pdf_to_md.py` — 通用提取脚本（所有 PDF 共用，禁止让 subagent 重复编写提取逻辑）
> - `WATERMARK.md` — 水印检测与清理专项指引
> - `QUICKREF.md` — 快速参考表（场景决策、常见问题）

## 使用场景

- 用户指定一个文件夹，希望把里面所有 PDF 批量转成 markdown
- 用户上传了一批 PDF，想快速获得每份 PDF 的完整内容（文本 + 表格）
- PDF 中包含大量表格（财报表格、数据表、统计表等），需要保留表格结构
- 后续需要对文本做进一步分析，但当前只需完成 PDF→MD 这一步

## 执行步骤

### 步骤 0：环境预检

```bash
"C:\Program Files\AlphaEngine\resources\python\python\python.exe" -c "import pdfplumber; print('ok')"
```

- 输出 `ok` → 环境完备，跳到步骤 1
- 报错 `ModuleNotFoundError` → 执行安装后继续：

```bash
"C:\Program Files\AlphaEngine\resources\python\python\python.exe" -m pip install pdfplumber --quiet
```

### 步骤 1：确定目标文件夹

扫描目标文件夹，获取所有 `.pdf` 文件列表（不区分大小写），按文件名排序。

向用户展示扫描结果：

```
找到 N 个 PDF 文件：

1. 文件名A.pdf（页面数：X）
2. 文件名B.pdf（页面数：Y）
3. ...

将提取到同目录下的 .md 文件中。
```

> 如果 PDF 数量 > 20，建议用户分批处理或筛选。

### 步骤 1.5：水印检测（自动进行，发现后询问）

读取每个 PDF 的前 3 页文本，检查是否有水印特征：
- 逐字分行散布的单字符（如「严」「密」「仅」「云」「基」等）
- 重复出现的纯水印单字行（如「参」「部」「内」「金」「基」「锋」「云」「供」「仅」）
- 竖排 Confidentiality 风格水印混入正文行间

**如果检测到水印**，主动询问用户：

> "检测到 PDF 带有水印，请提供水印文字内容（如'严格保密，仅供内部参考'），以便后续精准清理。如果水印内容复杂或不确定，可直接说'跳过'，我会提取后用规则清理。"

- 用户提供水印文字 → 记录水印内容，跳到步骤 2（按 WATERMARK.md 清理）
- 用户说"跳过" → 不清理，继续步骤 2（不阻断流程）

### 步骤 2：执行提取

#### 调度策略（防止模型限流）

| PDF 数量 | 策略 |
|----------|------|
| ≤ 2 | 直接调用通用脚本 |
| ≥ 3 | **分批调度**，每批最多 3 个，每批之间等待 5 秒 |

**分批执行流程**：
1. 将 PDF 列表按每批 3 个切分
2. 同一批内 **并行** 启动 subagent（每批最多 3 个）
3. 该批全部完成后，等待 5 秒，再启动下一批
4. 收集失败项，该批结束后统一重试（最多 2 次，间隔 30 秒）

#### 脚本路径

通用提取脚本：`{技能目录}/scripts/extract_pdf_to_md.py`

Python 路径：`C:\Program Files\AlphaEngine\resources\python\python\python.exe`

#### PDF 数量 ≤ 2：直接执行脚本

```bash
"C:\Program Files\AlphaEngine\resources\python\python\python.exe" "{技能目录}/scripts/extract_pdf_to_md.py" "{pdf_path}" "{output_dir}"
```

#### PDF 数量 ≥ 3：分批 subagent 并行

每批最多 3 个，在 **同一消息中并行发出所有 task 调用**。每个 subagent 的 prompt：

```
你负责调用通用脚本提取 PDF 为 markdown。

调用命令（直接在 bash 中执行，不要写 Python 代码）：
"C:\Program Files\AlphaEngine\resources\python\python\python.exe" "{技能目录}/scripts/extract_pdf_to_md.py" "{pdf_path}" "{output_dir}"

执行后解析输出：
- 成功：输出格式为 "OK|<md文件路径>|<页数>|<表格数>"
- 失败：输出格式为 "ERROR|<错误信息>"

完成后汇报：文件名、页数、表格数、md 文件路径（或失败原因）。
```

#### 失败重试逻辑

subagent 返回失败或限流错误时：
1. 等待 30 秒后重试该任务（最多 2 次）
2. 重试仍然失败 → 记录到失败列表，继续处理下一个
3. 全部完成后，如果失败列表非空，汇总展示并建议用户手动处理

### 步骤 3：展示汇总结果

```
✅ 提取完成！

| 序号 | PDF 文件 | 状态 | 页数 | 表格数 | MD 文件 |
|------|---------|------|------|--------|---------|
| 1 | 文件A.pdf | 成功 | 15 | 3 | 文件A.md |
| 2 | 文件B.pdf | 失败 | - | - | - |

输出目录：{文件夹路径}
```

失败项单独标注原因（密码保护 PDF、扫描件无文字等）。

## 输出格式

每个 .md 文件格式：

```markdown
# {文件名（不含 .pdf 后缀）}

> 来源：{原文件名}  |  共 {N} 页

---

--- 第 1 页 ---
{清理后的文本段落}

**表 1-1：**
| 列A | 列B | 列C |
| --- | --- | --- |
| 数据1 | 数据2 | 数据3 |

--- 第 2 页 ---
...
```

**表格格式化规则**：
- 第一行作为表头，第二行 `| --- | --- |` 分隔
- 单元格内换行替换为空格，保持单行
- 空行/全空表格跳过
- 编号格式：`表 {页码}-{该页表格序号}`

## 注意事项

### ⚠️ TRAPS（易错陷阱）

- **不重复安装**：步骤 0 已检查环境，任何情况下不要重复执行 `pip install pdfplumber`
- **不猜测路径**：用户没有明确指定文件夹时，先询问，不要假设路径
- **不覆盖已有文件**：同目录下已有同名 .md 时，必须询问用户是否覆盖
- **分批防超时**：超过 20 页必须分批读取，每批 10 页，页眉/页脚识别结果复用全局
- **水印先问再清**：步骤 1.5 检测到水印后，必须先询问用户再清理，不自行猜测水印内容
- **不合并**：所有 PDF 各自独立输出为独立 .md，不合并为一个文件
- **不分析**：纯原文提取，不做摘要、话题识别、结构化提取（那是 cross-talk-synthesis 的工作）
- **不重复造轮子**：subagent 必须调用通用脚本 `extract_pdf_to_md.py`，禁止在 prompt 中让 subagent 写 Python 提取逻辑
- **不擅自重试**：遇到限流错误时由主流程统一重试（间隔 30 秒），subagent 内部不要主动重试

### ℹ️ 技术说明

- **中文路径**：Python 3.13+ 原生支持中文路径，传入绝对路径字符串即可
- **表格不跨页合并**：每页表格独立编号，跨页表格暂不自动拼接
- **文本表格并存**：同一页中 `extract_text()` 和 `extract_tables()` 都会输出表格内容，这是 pdfplumber 正常行为，不做去重（原始文本行可在表格格式异常时作为兜底参考）
- **扫描件 PDF**：无文字层（全是图片）的 PDF，pdfplumber 返回空文本，跳过并标记为失败
- **并发控制**：每批最多 3 个 subagent，防止模型 API 限流
