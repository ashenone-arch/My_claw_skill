# 日度卖方热点选股

## 概述

识别当日卖方/机构集中关注的热点方向，直接流式输出结构化简报。**默认输出速览版**（Top10 + 热点话题 + 核心观点一句话），用户追问时再展开溯源。数据来源为 AlphaClaw 研报库 + iFinD 财经资讯，无需外部 API Token。

## 报告结构

1. **速览版（默认）** — 热点话题关键词 + Top10 表格 + 各方向核心观点一句话总结
2. **溯源版（按需）** — 用户追问"怎么看"或"展开"时，输出 Top10 的分析师观点摘要

## 使用场景

- 每日盘前/盘后快速了解卖方集中关注方向
- 行业定向热点扫描（"半导体今日热点"、"AI行业热点"）
- 发现机构抱团信号和共识强度
- 指定日期回溯历史热点

## 执行步骤

### 第一步：提取日期与行业定向

- 从用户消息中解析目标日期，默认"今天"
- 解析用户是否指定行业方向（如"半导体"、"AI"、"通信"），有则提取关键词用于检索过滤
- 支持指定日期（如"4月24日"、"2026-04-24"）

### 第二步：并行获取热点信号（按行业分组）

在同一条消息中并行发起检索，**按行业/主题分组而非全量检索**：

**按行业并行发起 3 个检索任务：**
- 用 `search_finance_reports` 搜索**当日行业点评**（report_types=["点评:行业点评", "点评:总量点评"]），date_range="past_day"，覆盖全市场行业
- 用 `search_finance_reports` 搜索**当日公司点评**（report_types=["点评:公司点评"]），date_range="past_day"
- 用 `search_finance_reports` 搜索**近 3 日会议纪要**（report_types=["会议纪要"]），date_range="past_week"

**若用户指定了行业关键词**（如"半导体"、"AI"、"通信"），在检索 query 中加入该关键词以过滤相关板块。

**若 iFinD MCP 工具可用**，并行调用获取当日热门财经资讯，提取高频关键词作为"热点话题"补充。

### 第三步：速览输出（默认，无需溯源）

从检索结果中提取所有股票，计算话题覆盖频次，按频次降序取 Top10，输出速览版报告：

```markdown
# 卖方热点选股 · YYYY-MM-DD（行业定向：XX）

> 数据来源：AlphaClaw研报库 | 生成时间：YYYY-MM-DD HH:MM

## 质检

> ✅ 数据完整，共检索到 N 篇研报/资讯
> ⚠️ 今日话题数量较少，可能受节假日影响（如话题数 < 5）

---
```

### 第四步：共识强度计算

| 覆盖券商数 | 共识标签 |
|-----------|---------|
| ≥8 家 | 🔴强共识 |
| 4-7 家 | 🟡中共识 |
| 2-3 家 | ⚪弱共识 |
| 1 家 | 🔘散点 |

### 第五步：溯源输出（用户追问时触发）

当用户追问以下情形时，触发溯源：
- "怎么看 XX"、"XX 公司首席观点"、"展开"
- "详细一点"、"给我溯源"

**溯源方式：按热度分组检索，而非逐个股检索**

将 Top10 按热度分成 3 组：
1. **高热组**（话题覆盖 ≥ 4 次）：用一次 `search_finance_reports` 打包查询这 2-3 只股票近 3 日分析师观点（report_types=["会议纪要:分析师纪要"]）
2. **中热组**（话题覆盖 2-3 次）：用一次 `search_finance_reports` 打包查询
3. **低热组**（话题覆盖 1 次）：暂时跳过，标注"暂无首席观点覆盖"

溯源输出格式：

## 注意事项

- 优先从 AlphaClaw 研报的 `company_name` 字段提取股票，辅以 iFinD 新闻中的公司名
- 同一话题中出现多只股票时，每只股票都计入频次
- **默认输出速览版**，溯源版仅在用户追问时触发，避免不必要的高频调用
- 溯源阶段按热度分组打包检索，**禁止逐个股串行检索**
- 行业 EDB 数据（`get_edb_data`）可作为热点行业趋势的补充验证
- **所有报告必须直接流式输出到对话中，禁止写入文件或使用 canvas**
- **不标注"积分不足"等 Gangtise 特定内容**
