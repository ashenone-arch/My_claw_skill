# 日度卖方热点选股

## 概述

识别当日卖方/机构集中关注的热点方向，输出结构化简报：质检 → 热点个股 Top20 → 个股详细溯源（首席观点原文）。数据来源为 AlphaClaw 研报库 + iFinD 财经资讯，无需外部 API Token。

## 报告结构

1. **质检区块** — 数据完整性告警
2. **热点个股速览（Top20）** — 根据话题/研报覆盖频次排名，含昨日对比
3. **个股详细溯源** — Top20 个股的分析师观点摘要

## 使用场景

- 每日盘前/盘后了解卖方集中关注方向
- 发现机构抱团信号和共识强度
- 追踪热点个股的分析师覆盖变化
- 指定日期回溯历史热点

## 执行步骤

### 第一步：提取日期

从用户消息中解析目标日期。默认"今天"，支持指定日期（如"4月24日"、"2026-04-24"）。

### 第二步：并行获取热点信号

在同一条消息中并行发起检索：

**组 A — 热点资讯（iFinD）：**
- 用 `search_trending_news` 获取当日热门财经资讯（设置 `time_scope="24小时"`），提取高频关键词作为"热点话题"
- 用 `search_news` 获取当日公司相关新闻

**组 B — 卖方研报（AlphaClaw）：**
- 用 `search_finance_reports` 搜索当日行业点评（report_types=["点评:行业点评", "点评:总量点评"]），date_range="past_day"，识别卖方集中讨论的方向
- 用 `search_finance_reports` 搜索当日公司点评和业绩点评（report_types=["点评:公司点评"]），date_range="past_day"
- 用 `search_finance_reports` 搜索近 3 日会议纪要（report_types=["会议纪要"]），date_range="past_week"

### 第三步：提取股票频次 + 生成 Top20

从上一步的检索结果中：
1. 提取所有被提及的 A 股股票代码和名称
2. 统计每只股票出现在多少篇资讯/研报中（即"话题覆盖次数"）
3. 按频次降序排列，取 Top20
4. 若用户指定了对比日期（如"和昨天比"），同时解析昨日数据做频次对比

### 第四步：获取 Top20 的首席观点

对 Top20 中的每只股票，用 `search_finance_reports` 搜索近 3 日分析师观点（report_types=["会议纪要:分析师纪要"]），按股票聚合返回：
- 观点数量、覆盖券商数、覆盖首席数
- 前 3 条代表性观点的标题和摘要

### 第五步：计算共识强度

| 覆盖券商数 | 共识标签 |
|-----------|---------|
| ≥8 家 | 🔴强共识 |
| 4-7 家 | 🟡中共识 |
| 2-3 家 | ⚪弱共识 |
| 1 家 | 🔘散点 |

### 第六步：渲染并交付报告

按 `references/output_format.md` 的格式生成报告，包含：
- 质检区块（覆盖完整性提示）
- 热点个股 Top20 表格
- 个股详细溯源（分析师观点原文）

将报告保存为 Markdown 文件，向用户交付。

## 注意事项

- 优先从 AlphaClaw 研报的 `company_name` 字段提取股票，辅以 iFinD 新闻中的公司名
- 同一话题中出现多只股票时，每只股票都计入频次
- 昨日对比数据：如果用户没有明确要求对比，可以省略以节省检索成本
- 如果某只 Top20 股票没有分析师观点覆盖，在溯源中标注"暂无首席观点覆盖"
- 行业 EDB 数据（`get_edb_data`）可作为热点行业趋势的补充验证
